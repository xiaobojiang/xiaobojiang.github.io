In the early period of computer science, the world has only one encoding: ASCII.

ASCII, stands for American Standard Code for Information Interchange. It is a 7-bit character code where each individual bit represents a unique character. In contains common Engligh characters and symbols

Unicode

When the computer usage scales up, non-English regions cannot use their own characters and symbols, and multi-lingual is a mandatory.

Then The Unicode Consortium develops the Unicode Standard. Their goal is to replace the existing character sets with its standard Unicode Transformation Format (UTF).

In the Unicode system, each single character, Chinese or Janpanese, can have a unique Unicode point.

UTF-8

UTF-8 encodign is co-invented by Golang creator Ken Thompson and Rob Pike, now is the standard of Unicode.

A character in UTF8 can be from 1 to 4 bytes long. UTF-8 can represent any character in the Unicode standard.

Quite interestingly, by the special design, UTF-8 is backwards compatible with ASCII.

If the highest bit is 0, this means it is a ASCII char.

If the highest bits starts with 110, which means it is 2 bytes long character, the following 1 byte start with bits 10

If the highest bits starts with 1110, which means it is 3 bytes long character, the following 2 bytes start with bits 10

If the highest bits starts with 11110, which means it is 4 bytes long character, the following 3 bytes start with bits 10

Its encoding is designed like:

0xxxxxxx                             runes 0-127    (ASCII)

110xxxxx 10xxxxxx                    128-2047       (values <128 unused)

1110xxxx 10xxxxxx 10xxxxxx           2048-65535     (values <2048 unused)

11110xxx 10xxxxxx 10xxxxxx 10xxxxxx  65536-0x10ffff (other values unused)

all the "xxx" bits combined can be unicode representation "\u4e16" (16 bits) or "\U00004e16" (32bits)

Take Chinese word "世" as example, its unicode representation is "\u4e16" (16 bits representation)

Its full hex value in 32 bits is "\xe4\xb8\x96"

1110 0100   1011 1000  1001 0110  1101 0111  1001 0101 1000 1100 (hex value string)

         0100       11 1000    01 0110 (get the xxx bits)

         0100  1110 0001 0110     (combined)

         4   e    1    6 (unicode representation)

Alright, that's the simple journey into UTF-8.
