When working with Python in the command line, it’s easy to overlook presentation.

But a well-structured and visually appealing output can dramatically improve how you read, debug, and communicate information — especially when working with prompts, structured text, or configuration data.

The rich library makes this easier than ever.

In this post, we’ll walk through a neat snippet that turns dull terminal output into a color-coded, panelized display, and explore some practical use cases.

🧩 The Problem: Plain Text Fatigue

When you’re iterating on AI prompts, debugging XML templates, or viewing nested structures, standard print() statements don’t cut it.

A raw output might look like this:

<prompt>

##User Input

Hello, world!

###[Context]

This is a test.

</prompt>

Readable? Kind of.

Visually engaging or structured? Not at all.

Let’s fix that.

💻 The Solution: rich Formatting

Here’s a simple function that brings color, structure, and polish to your terminal output:

from rich.console import Console

from rich.panel import Panel

from rich.text import Text

console = Console()

def show_prompt(prompt_text: str, title: str = "Prompt", border_style: str = "blue"):

    """Display a prompt with rich formatting and XML tag highlighting.

    Args:

        prompt_text: The prompt string to display

        title: Title for the panel (default: "Prompt")

        border_style: Border color style (default: "blue")

    """

    # create a formatted diaplay of the prompt

    formatted_text = Text(prompt_text)

    formatted_text.highlight_regex(r"<[^>]+>", style="bold blue")  # Highlight XML tags

    formatted_text.highlight_regex(

        r"##[^#\n]+", style="bold magenta"

    )  # Highlight headers

    formatted_text.highlight_regex(

        r"###[^#\n]+", style="bold cyan"

    )  # Highlight sub-headers

    #Display in a panel for better presentation

    console.print(

        Panel(

            formatted_text,

    title=f"[bold green]{title}[/bold green]",

            border_style=border_style,

            padding=(1, 2),

      )

)

🔍 Breaking It Down

1. Highlighting Key Elements

The function uses regular expressions with highlight_regex() to colorize specific text patterns:

r"<[^>]+>" → highlights XML-style tags in bold blue

r"##[^#\n]+" → marks section headers (##Header) in magenta

r"###[^#\n]+" → colors subheaders (###Subheader) in cyan

This instantly makes structured text pop visually.

2. Wrapping in a Panel

The Panel component from rich.panel adds:

  • A border (color-customizable)

  • A title (styled in green)

  • Padding around the content for clean spacing

This simple container elevates the presentation, making it feel like an output block in a UI rather than raw text.

3. Console Output

Finally, the function uses console.print() to render the styled panel.

You can easily adjust the border_style (e.g., "yellow", "red", "cyan") or change fonts and highlights depending on your data.

🧠 Example in Action

Here’s how it looks in use:

prompt = """

<prompt>

##User Input

What is the capital of France?

###[Context]

You are a helpful assistant.

</prompt>

"""

show_prompt(prompt, title="LLM Prompt Example", border_style="cyan")

Your eyes can instantly distinguish structure, hierarchy, and boundaries.

🧰 Real-World Applications

This approach can add value across multiple scenarios:

1. Prompt Engineering

When developing large language model (LLM) prompts that include structured sections or metadata, use this to visualize templates clearly during debugging or demonstration.

2. Configuration Display

Display configuration files, JSON snippets, or YAML content with colored keys and values.

It makes it much easier to spot errors or version differences at a glance.

3. CLI Utilities

If you’re building a command-line interface (CLI) tool using frameworks like typer or click, this adds polish and clarity to your outputs — turning logs or summaries into visually guided sections.

4. Log Review and Report Formatting

In data pipelines or automation scripts, highlight important events or summaries in color-coded panels (e.g., “✅ SUCCESS”, “⚠️ WARNING”, “❌ ERROR”).

🧾 Final Thoughts

The rich library proves that terminal output doesn’t have to be boring.

With a few lines of code, you can add structure, emphasis, and style — making your CLI tools more readable and professional.

Whether you’re displaying LLM prompts, configs, or debugging complex strings, rich helps you and your users see the data in a whole new light.
