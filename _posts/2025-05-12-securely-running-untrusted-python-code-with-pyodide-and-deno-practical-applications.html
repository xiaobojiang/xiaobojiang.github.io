In today’s digital landscape, executing untrusted Python code safely is crucial for applications like AI agents, educational platforms, and online code execution services. Combining Pyodide and Deno offers a robust solution:

  • Pyodide compiles Python to WebAssembly (WASM), enabling Python execution in a browser or JavaScript environment.

  • Deno is a secure JavaScript/TypeScript runtime that provides fine-grained permission controls and a modern development experience.

Together, they allow for executing untrusted Python code in a sandboxed environment, ensuring safety and isolation.

🎯 Real-World Applications

1. AI Agents and Chatbots

AI agents often need to execute dynamically generated Python code. Using Pyodide within Deno ensures that this code runs in a secure, isolated environment, preventing potential security risks. For instance, Pydantic’s mcp-run-python server facilitates this by providing a sandboxed execution environment.

2. Educational Platforms

Online coding platforms can allow students to write and execute Python code without risking the host system’s integrity. By leveraging Pyodide and Deno, these platforms can offer real-time code execution in a browser, enhancing the learning experience.

3. Online Code Execution Services

Services that allow users to run Python snippets online can benefit from this setup. It ensures that each code execution is sandboxed, preventing malicious code from affecting other users or the server.

🔧 Implementing a Pyodide Sandbox with Deno

For a lightweight and customizable approach, integrating Pyodide directly within a Deno script is effective. Simon Willison provides an insightful example:

import pyodideModule from "npm:pyodide/pyodide.js";
import { readLines } from "https://deno.land/std@0.186.0/io/mod.ts";

const pyodide = await pyodideModule.loadPyodide();

for await (const line of readLines(Deno.stdin)) {
    let input;
    try {
        input = JSON.parse(line);
    } catch (error) {
        console.log(JSON.stringify({ error: "Invalid JSON input: " + error.message }));
        continue;
    }

    if (input.shutdown) {
        break;
    }

    try {
        const result = await pyodide.runPythonAsync(input.code || "");
        console.log(JSON.stringify({ output: result }));
    } catch (error) {
        console.log(JSON.stringify({ error: error.message.trim().split('\n').pop() || '' }));
    }
}

This script reads Python code from standard input, executes it using Pyodide, and outputs the result. It’s a straightforward method to sandbox Python execution without external dependencies.

🧰 Leveraging LangChain’s PyodideSandbox

For applications requiring stateful execution and integration with AI workflows, LangChain’s PyodideSandbox offers a higher-level abstraction:

from langchain_sandbox import PyodideSandbox


async def main():
    # Create a PyodideSandbox instance
    sandbox = PyodideSandbox("./sessions", allow_net=True)

    # Code to execute
    code = """
    import numpy as np
    x = np.array([1, 2, 3])
    print(x)
    """

    # Execute the code in the sandbox
    result = await sandbox.execute(code, session_id="123")
    print(result.stdout)

import asyncio 
asyncio.run(main())

This setup allows for persistent sessions, enabling variables and imports to be maintained across executions. It’s particularly useful for interactive environments and AI agents.

🤖 Deploying Pydantic’s MCP Run Python Server

Pydantic’s mcp-run-python server provides a ready-to-use solution for executing Python code securely:

  1. Run the Server:

deno run -N -R=node_modules -W=node_modules --node-modules-dir=auto jsr:@pydantic/mcp-run-python stdio

  1. Integrate with PydanticAI:

from mcp import ClientSession, StdioServerParameters
from mcp.client.stdio import stdio_client

code = """
import numpy
a = numpy.array([1, 2, 3])
print(a)
a
"""
server_params = StdioServerParameters(
    command='deno',
    args=[
        'run',
        '-N',
        '-R=node_modules',
        '-W=node_modules',
        '--node-modules-dir=auto',
        'jsr:@pydantic/mcp-run-python',
        'stdio',
    ],
)


async def main():
    async with stdio_client(server_params) as (read, write):
        async with ClientSession(read, write) as session:
            await session.initialize()
            tools = await session.list_tools()
            print(len(tools.tools))
            #> 1
            print(repr(tools.tools[0].name))
            #> 'run_python_code'
            print(repr(tools.tools[0].inputSchema))
            """
            {'type': 'object', 'properties': {'python_code': {'type': 'string', 'description': 'Python code to run'}}, 'required': ['python_code'], 'additionalProperties': False, '$schema': 'http://json-schema.org/draft-07/schema#'}
            """
            result = await session.call_tool('run_python_code', {'python_code': code})
            print(result.content[0].text)
            """
            <status>success</status>
            <dependencies>["numpy"]</dependencies>
            <output>
            [1 2 3]
            </output>
            <return_value>
            [
              1,
              2,
              3
            ]
            </return_value>
            """

import asyncio
asyncio.run(main())

This approach is ideal for integrating Python execution into AI agents, offering features like automatic dependency management and detailed error reporting.

⚠️ Considerations

  • Security: While these methods provide isolation, it’s essential to understand that the security relies on the robustness of Deno’s runtime and Pyodide’s implementation. Always assess the risk based on your application’s requirements.

  • Performance: Using WebAssembly introduces some overhead. For performance-critical applications, evaluate the trade-offs between security and execution speed.

  • Limitations: Certain Python modules, especially those requiring native extensions, may not be supported in Pyodide. Ensure that the necessary libraries are compatible with Pyodide’s environment.

By leveraging Pyodide and Deno, you can create a secure and efficient sandbox for executing Python code, tailored to your specific needs and constraints.
