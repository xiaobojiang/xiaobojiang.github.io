Recently, I moved my wordpress server from one cloud vendor to another.

I copied all of my website's data in mysql database, and data in mounted folder of wordpress container image.

Rebuild the services on the server is not that difficult, I made some ansible script to help me automated these things, includeing let's encrypt certificates & regular renew job, nginx configuration, docker installation and start service of mysql and wordpress.

Restoring data is also simple, load the dump sql into mysql, and unzip the html folder to new server, and mount it new wordpress.

After firing up, the website looks good, it can load and view.

Then I noticed in my admin, I have few updates for plugin and themes. I found that I cannot just click and upgrade themes It always asked me about ftp address and user name & password.

It is annoying, just to upgrade the theme, why shall I setup some ftp for this.

Furthermore, in the new post, uploading the new pic timed out.

I did some search on the internet. The results points to the problem of file permission.

The unziped data in the folder has owner of root and owner group of root, this leads to wordpress cannot modify files in the folder.

If you also encounter similar issue, here is the quick remedy:

1, change wp-content folder in html directory with permission 777

$ chomod 777 -R html/wp-content

2, create a new post, and upload a picture.

In the html/wp-content/uploaded/<year>/<month>, you can find the image.

Check the file owner and owner group, with following command

$ stat -c '%u %G' <filename>

From the output, my wordpress is using "33" as owner name, and group is "tape"

3, change the whole html folder with this user name and group: 

$ chown -R 33:tape html

All steps done! Now you shall can already update themes and plugins.

Now the admin page is updated with latest themes and plugins. Great!
