In Kubernetes, managing deployments effectively means understanding the exact container images running in your environment. This information is crucial for security, troubleshooting, and upgrading applications. Fortunately, kubectl—the Kubernetes command-line tool—offers a quick way to retrieve deployment details, including container image names and tags, all within a specific namespace.

Why Extract Container Image Tags?

Knowing which versions (tags) of container images are being used across your deployments helps ensure that:

  • Version Control: You’re running the expected version of an application.

  • Security: You can identify and address vulnerabilities tied to specific image versions.

  • Troubleshooting: Pinpoint issues that may be related to a particular image or tag.

  • Upgrades/Rollbacks: Manage application lifecycle changes more effectively.

Extracting Container Image Information with 

kubectl

To retrieve deployment names along with their associated container image names and tags, you can use a simple kubectl command with jsonpath formatting. Here’s the basic command:

kubectl get deployments -n <namespace> -o=jsonpath='{range .items[]}{.metadata.name}{" -> "}{.spec.template.spec.containers[].image}{"\n"}{end}'

Command Breakdown:

  • kubectl get deployments: Retrieves a list of all deployments in your cluster.

  • -n <namespace>: Specifies the namespace you want to query. Replace <namespace> with your desired namespace name.

  • -o=jsonpath='{...}': This flag formats the output using JSONPath, which allows you to extract specific fields from the returned JSON object.

    • {range .items[*]}: Loops through all deployment items returned in the response.

    • {.metadata.name}: Extracts the name of each deployment.

    • {" -> "}: Adds a separator between the deployment name and the container image.

    • {.spec.template.spec.containers[*].image}: Retrieves the container image(s) used in each deployment, including the version tag (e.g., nginx:1.21).

    • {"\n"}: Adds a newline after each output to ensure clarity and separation between deployments.

  • {end}: Marks the end of the iteration.

Example Output

Assume you’re interested in the prod namespace. Running the command would return output like this:

kubectl get deployments -n prod -o=jsonpath='{range .items[]}{.metadata.name}{" -> "}{.spec.template.spec.containers[].image}{"\n"}{end}'

Output:

my-app-deployment -> my-app:1.3.0

nginx-deployment -> nginx:1.21.4

redis-deployment -> redis:6.2.5

In this example, you see the deployment name followed by the container image and tag, which is extremely useful for auditing and managing your deployments.

Customizing Your Query

While the above query works in most cases, you can customize it based on your needs. Here are a few modifications you might find useful:

Filter for a Specific Deployment

If you need to retrieve image information for a specific deployment, you can narrow your query:

kubectl get deployment <deployment-name> -n <namespace> -o=jsonpath='{.spec.template.spec.containers[*].image}'

For example:

kubectl get deployment my-app-deployment -n prod -o=jsonpath='{.spec.template.spec.containers[*].image}'

This will only show the container image(s) used by the my-app-deployment.

Handle Multiple Containers in a Deployment

If your deployment contains multiple containers, the command will list all of them. For example, if you have a deployment with sidecar containers, this will capture all the images:

kubectl get deployments -n <namespace> -o=jsonpath='{.items[].metadata.name}{" -> "}{.spec.template.spec.containers[].image}'

Add More Fields to the Output

You can extend the jsonpath query to include additional deployment details, such as labels. This is useful for adding context to the image information:

kubectl get deployments -n <namespace> -o=jsonpath='{range .items[]}{.metadata.name}{" -> "}{.spec.template.spec.containers[].image}{" -> "}{.metadata.labels}{"\n"}{end}'

This will show the deployment names, images, and associated labels.

Conclusion

Using kubectl with JSONPath is an efficient way to extract detailed information about the container images running in your Kubernetes deployments. Whether you’re performing an audit, upgrading applications, or troubleshooting, this command helps ensure that you’re using the correct container versions across your environments.

Next time you need to check container image tags in your deployments, simply use the command outlined above, and you’ll have a clear overview of your Kubernetes applications in no time.
