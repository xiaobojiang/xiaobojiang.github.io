Setting up password protection for a website served by Nginx can be achieved using the HTTP Basic Authentication method. Here's a step-by-step guide for implementing password protection on your Nginx server:

Step 1: Create a Password File

  1. Open a terminal on your server.

  2. Use the htpasswd command to create a password file. Replace username with the desired username. You will be prompted to enter and confirm the password.

sudo htpasswd -c /etc/nginx/.htpasswd username

The -c flag is used to create a new file.

Step 2: Edit Nginx Configuration

  • Open the Nginx configuration file for your website. This is typically located in the /etc/nginx/sites-available/ directory.

sudo nano /etc/nginx/sites-available/your-site

Replace your-site with the actual configuration file name for your website.

  • Add the following lines inside the server block, adjusting the paths as needed

server {
    # ... other server configurations ...

    location / {
        auth_basic "Restricted Access";
        auth_basic_user_file /etc/nginx/.htpasswd;
        # ... other location configurations ...
    }

    # ... other server configurations ...
}

The auth_basic directive sets the authentication realm, and auth_basic_user_file specifies the path to the password file created in Step 1.

  • Save the changes and exit the text editor.

Step 3: Test Nginx Configuration

  • Ensure there are no syntax errors in your Nginx configuration:

sudo nginx -t

  • If the test is successful, reload Nginx to apply the changes:

sudo systemctl reload nginx

Step 4: Access the Protected Site

  1. Visit your website in a web browser.

  2. You will be prompted to enter the username and password you set up in Step 1.

Additional Notes:

  • If you want to protect multiple locations within your site, you can add the authentication directives to each location block as needed.

  • Ensure that the password file (/etc/nginx/.htpasswd in this example) is stored in a secure location, and its permissions are appropriately restricted.

  • Always backup your Nginx configuration before making changes.

This basic authentication method provides a simple way to secure your site, but it's worth noting that the credentials are transmitted in base64 encoding, which is not considered secure on its own. For enhanced security, consider using HTTPS along with basic authentication.
