Home Directory

After Jenkins installation, one of the next big thing is: Home directory

We may have to rely on home directoy, when:

  • Our script uses '~' as shorthand of home directory
  • Configure keys for remote ssh
  • Configure setting.xml for maven in ~/.m2

There is a trap here, the Jenkins's default home directory is not our login users to Ubuntu.

It is specified in /etc/default/jenkins

By default JENKINS_HOME is located at /var/lib/jenkins

and default user and user group is jenkins.

There are posts in the internet, who wants to reuse login home directory and user for Jenkins.

Thus change the configuration of JENKINS_USER, JENKINS_GROUP and JENKINS_HOME, and chown /var/run/jenkins/log /var/lib/jenkins with lgoin user.

Upper is not working in new version Jenkins, eg 2.370. Jenkins cannot startup, and it will mess up your new user’s home directory.

Recommended way about Jenkins home directory issue:

1, Still use Jenkins as running user, do not modify /etc/default/jenkins in Debian

2, Always remember that, in Jenkins, the $HOME or $JENKINS_HOME is pointed to /var/lib/jenkins, not your pc user’s /home/username, and user is jenkins

3, if you have ssh keys want reuse for git in Jenkins

Copy the keys from /home/username/.ssh to /var/lib/jenkins/.ssh

And change ownership to jenkins

$sudo chown -R jenkins:jenkins /var/lib/jenkins/.ssh

Maven Configuration

use your own maven setting xml

copy the file to Jenkins home directory

$ cp ~/.m2/setting.xml /var/lib/jenkins/.m2/

and change ownership:

$sudo chown -R jenkins:jenkins /var/lib/jenkins/.m2/setting.xml

Use aliyun maven repository, add following section in setting.xml:

     <mirror>
       <id>aliyunmaven</id>
       <mirrorOf>*</mirrorOf>
       <name>ali</name>
       <url>https://maven.aliyun.com/repository/central</url>
     </mirror>

Issue when using maven central repository:

“Failed to read artifact descriptor for org.apache.maven.plugins:maven-clean-plugin:jar:2.5: Could not transfer artifact org.apache.maven.plugins:maven-clean-plugin:pom:2.5 from/to central”

Override ssl check, by add below line in maven command:

-Dmaven.wagon.http.ssl.insecure=true -Dmaven.wagon.http.ssl.allowall=true

works for me.
