What are magic commands?

  • When you use a Jupyter Notebook (or its underlying IPython kernel), you have access to special “magic” commands. 

  • Magic commands start with % or %%.

    • Line magics (single-line) start with % and apply only to that line.

    • Cell magics (multi-line) start with %% and apply to the whole cell. 

  • They make common tasks easier and faster — e.g. timing code, loading files, debugging, running shell commands — without leaving your notebook. 

🔧 Some of the Most Useful Magic Commands (and what they do)

There are many more — line and cell magics — but these give a good idea of the power and convenience magics add. 

📚 Simple Example: Why Magic Commands Help

Imagine you’re analyzing some data, doing calculations, and plotting the results inside a notebook. With magic commands, your workflow can look like this:

# 1. Make plots show up inside the notebook
%matplotlib inline
import matplotlib.pyplot as plt
import numpy as np

# 2. Create some data and plot
x = np.linspace(0, 10, 200)
y = np.sin(x)
plt.plot(x, y)
plt.title("Sine wave")
plt.show()

# 3. Check how fast a small calculation runs
%time result = sum([i**2 for i in range(1000000)])

# 4. Try a faster approach and benchmark
%timeit sum(i**2 for i in range(1000000))

# 5. Load external helper code from a file, if you have one.
# %load helper_functions.py

# 6. Run a full script with more code
# %run data_preprocessing.py

# 7. When you're done & want to clean up workspace:
# %reset -f

This way, you can visualize, profile performance, run external code, and manage your workspace — all without leaving the interactive notebook environment.

✅ Why Use Magic Commands — Key Benefits

  • They save time and keystrokes compared to writing full Python or shell code for common tasks.

  • They make experiments, testing and debugging faster and easier.

  • They keep everything inside the notebook — code, plots, debugging, external scripts — making your work well organized and reproducible.

  • They are especially helpful when working with data, plotting, or trying different code approaches rapidly.
