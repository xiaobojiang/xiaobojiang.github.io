If you have frequent docker build on a linux machine, and you found that this machine's hard disk space has consumed to a limit.

While you tried docker image prune, docker container prune, but no space can be collected.

Then you may use Docker’s disk usage command to understand where space is being consumed:: docker system df

If you observe the result that the majority space is occupied by:

Build Cache    983      0        128.6GB  128.6GB

Do not worry, The Build Cache in Docker refers to cached layers created during image builds. These layers are stored to speed up future builds but can accumulate over time and consume significant disk space.

To manage and free up space used by the Docker build cache, follow this step:

To remove all unused build cache, run:

$ docker builder prune

By default, this removes all build cache that is not currently being used.

After a while, the disk will be cleaned and okay to go for next build!.

Or Limit Cache Retention:

To manage build cache retention effectively, consider limiting its size or lifespan:

• Specify the --filter option with until to remove cache older than a specific duration:

$docker builder prune --filter "until=24h"
