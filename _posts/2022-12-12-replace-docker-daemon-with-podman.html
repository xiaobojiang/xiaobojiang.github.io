Docker daemon is the starting point that I learn and use container technology.

It is good and powerful.

Later, in enterprise deployment scenario, I faced one security challendge, that is docker daemon by default runs with root access. Ah, that is not good.

There are few ways[1,2] to make docker daemon run in root-less mode, however, it is hardcore and experimental.

Later, I found a nice alternative: Podman

"Podman is a daemonless, open source, Linux native tool designed to make it easy to find, run, build, share and deploy applications using Open Containers Initiative (OCI) Containers and Container Images."

It is daemon-less, and default it runs in root-less mode, you can start podman with non-root use.

You may also found in some old posts, podman cannot build image.

Now, podman also support docker image build, it is more and more powerful now and good replacement of container runtime.

You can follow the guide to install podman[3].

You may found in latest redhat 8 distribution, podman is default added.

However, in some old linux ditributions, you cannot find podman in installation source.

From podman website, it is easy to install from package source in: RHEL 8+, Debian 11+, Ubuntu 20.10+

Docker cli emulator

Here, we wrote a lot of scripts with docker as cli command, eg docker run. if we switch to podman, we can use emulator to keep script same.

One manual way is

$ alias docker=podman

to make it permanent, we can add this line in bashrc file.

Another way is install this package: podman-docker

$ yum install podman-docker

It will do the same work.

Now, check with any docker command:

[root@deploy ~]# docker ps
Emulate Docker CLI using podman. Create /etc/containers/nodocker to quiet msg.
CONTAINER ID  IMAGE  COMMAND  CREATED  STATUS  PORTS  NAMES

works, cool!
