SCP (secure copy) command is used to copy files between servers in a secure way.

It used the same authentication and security as it is used in SSH (security shell)

Full syntax:

scp [-346BCpqrTv] [-c cipher] [-F ssh_config] [-i identity_file] [-l limit] [-o ssh_option] [-P port] [-S program] [[user@]host1:]file1 … [[user@]host2:]file2

  • cp –P port: Specifies the port to connect on the remote host.
  • scp –p: Preserves modification times, access times, and modes from the original file.
  • scp –q: Disables the progress meter.
  • scp –r: Recursively copy entire directories.
  • scp –S program: Name of program to use for the encrypted connection. The program must understand ssh(1) options.
  • scp –v: Verbose mode. Causes scp and ssh to print debugging messages about their progress. This is helpful in debugging connection, authentication, and configuration problems.
  • scp -f: if there is file with same name in the destination, then it will be deleted before copy action

Several typical usage:

Copy single file from local to remote server

$ scp ~/Download/sample.txt root@192.168.100.100:/file/path

or -i with ssh private key

$ scp -i ~/.ssh/iotolon.pem ~/Download/sample.txt root@192.168.100.100:/file/path

Copy single file from remote server to local

$ scp  root@192.168.100.100:/file/path/sample.txt ~/Download/file/path

Or provided with ssh private key 

$ scp -i ~/.ssh/iotolon.pem  root@192.168.100.100:/file/path/sample.txt ~/Download/file/path

Copy a folder, add -r

Eg, copy a folder from local to remote server

$ scp -r ~/Download/folder root@192.168.100.100:/file/path
