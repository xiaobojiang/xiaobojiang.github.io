If you are managing your libs with uv, and create a virtual env for your jupyter notebook. Then you face the issue that notebook cannot discover the python env as runtime. Then just follow this simple guide to make it work!

First create venv from new project

$ uv venv

or sync libs from existing project

$ uv sync

then activate the virtual env

$ source .venv/bin/activate

just “activating” the venv in your shell doesn’t make Jupyter automatically use it. Jupyter notebooks use kernels, and you need to register your uv venv as one. Here’s the step-by-step flow:

1. Make sure your uv venv has ipykernel

From inside your project:

uv pip install ipykernel

2. Register the venv as a Jupyter kernel

Still inside the project (with uv managing the venv):

uv run python -m ipykernel install --user \

    --name=uv-deep-agents \

    --display-name "Python (uv: deep-agents)"

  • --name is the internal kernel name (no spaces).

  • --display-name is what you’ll see in the Jupyter UI.

3. Switch to the kernel in your notebook

In Jupyter (Notebook or Lab):

  • Menu → Kernel → Change kernel → pick Python (uv: deep-agents).

4. Verify it’s using the uv venv

Run inside a cell:

import sys

print(sys.executable)

It should point to the .venv/bin/python (or wherever uv created the environment).

you will get something like this:

👉 After this, your notebook will import everything you installed with uv sync.
