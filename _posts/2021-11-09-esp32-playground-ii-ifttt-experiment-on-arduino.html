What is IFTTT?

If This Then That (commonly known as IFTTT, /ɪft/), is a web-based service that allows users to create chains of conditional statements triggered by changes that occur within other web services. It is both a website and a mobile app of free service with the following slogan: "Put the Internet to work for you".

IFTTT aims to help people use the open API of various websites to monitor the triggers set by users. If triggers are triggered, actions set by users will be executed. Actions can be email, sms, or even write a blog. Usually, we can create n applets to meet our various automation needs.

Prerequisite of experiment

Before this experiment, something needs to be prepared:

  • One Dfrobot ESP32-E board
  • Arduino IDE
  • IFTTT account

Configure Arduino IDE

Some extra configuration are required for Arduino IDE.

First please follow previous post to install ESP32 boards in Arduino IDE.

Second you need to install “ArduinoJson” library, this library helps you to compose json object. You can install it in menu: Sketch -> Include Library -> Manage Library. In the library manager, search “ArduinoJson”, and select the one whose author is Benoit and install.

Figure 3. Install ArduinoJson

Configure IFTTT

1, enter IFTTT website (ifttt.com), login with your account. If you do not have one, you can register one in short time.

Figure 4. Create new applet

2, Click “create” to make a new applet.

Figure 5. main elements in a new applet

3, Follow the guide, we need to define the “If this”, which is the event. Let us click add, search and find the “webhooks”. Check more details of webhook in : https://ifttt.com/maker_webhooks

Figure 6. choose webhooks as event

Then select “Receive a web request”.

Figure 7. select "Receive a web request" in webhooks

Provide this event with a name, like “button_pressed”, then click “Create trigger”

Figure 8. input a name for event

With all these settings, you can make a POST or GET web request to trigger the event:

https://maker.ifttt.com/trigger/{event_name}/with/key/{your_key}

With an optional JSON body of:

{ "value1" : "", "value2" : "", "value3" : "" }

The data is completely optional, and you can also pass value1, value2, and value3 as query parameters or form variables. This content will be passed on to the action in your Applet.

4, We also need to define “Then That”, which is the action to trigger.

Choose “Email” as the action service.

Figure 9. select email as action service

Click next step.

Here, if we have not registered any email address before. A pop window will ask input for email address, pin code to valid your mail box. Just follow the guide, those pics will be skipped here.

Figure 10. select "Send me an email"

We can customize the email subject and body text, in which “EventName, Value1, Value2, Value3” come from the webhook event request.

Figure 11. customize email template

5, click the finish to complete the whole applet, its status is connected. We can drag the radio button to connect or disconnect this applet. If this applet is not desired, click Archive button to disable it.

Figure 12. a complete applet with event and action defined

Cofiguration of IFTTT is done, now if we make a web request to IFTTT event url, a mail with defined subject and payload will be sent to our mail box.

Coding and flash with Arduino

Coding part would be simple, the program first provides WiFi SSID and password to ESP32 to make it connected to AP, then listening for the button pressed (pin27), finally, if button pressed, it will make a post request to IFTTT webhook url.

Following is the whole code

#include <WiFi.h>
#include <HTTPClient.h>
#include <ArduinoJson.h>

#define BUTTON_PIN  D4

const char* WIFI_SSID     = "ssid";
const char* WIFI_PASSWORD = "password";
const char* ntpServer = "pool.ntp.org";
const long gmtOffset_sec = 28800;
const int daylightOffset_sec = 0;

String IFTTT_EVENT   = "button_pressed";
String IFTTT_KEY     = "my_key";

String IFTTT_VALUE_1 = "hi";
String IFTTT_VALUE_2 = "greeting";
String IFTTT_VALUE_3 = "0";

HTTPClient http;
unsigned long lastTime = 0;
unsigned long timerDelay = 10000;
DynamicJsonDocument doc(1024);

void setup() {
  // put your setup code here, to run once:
  Serial.begin(115200);
  pinMode(BUTTON_PIN, INPUT_PULLUP);
  WiFi.begin(WIFI_SSID, WIFI_PASSWORD);
  Serial.println("Connecting");
  while(WiFi.status() != WL_CONNECTED)
  {
    delay(500);
    Serial.print(".");
  }
  Serial.println("");
  Serial.print("wifi connect success");
  configTime(gmtOffset_sec, daylightOffset_sec, ntpServer);
}

unsigned int lastState = HIGH;
unsigned int currentState;

void loop() {
  // put your main code here, to run repeatedly:
  currentState = digitalRead(BUTTON_PIN);

  if(lastState == HIGH && currentState == LOW)
  {
    //pressed
    String url = "https://maker.ifttt.com/trigger/"+IFTTT_EVENT+"/with/key/"+IFTTT_KEY;
    doc["value1"] = IFTTT_VALUE_1;
    doc["value2"] = IFTTT_VALUE_2;
    doc["value3"] = IFTTT_VALUE_3;
    String json;
    serializeJson(doc, json);
    Serial.println(url);
    Serial.println(json);
    http.begin(url);
    http.addHeader("Content-Type","application/json");
    int httpCode = http.POST(json);
    if (httpCode == HTTP_CODE_OK)
    {
      Serial.println("IFTTT triggered!");
    }
    else
    {
      Serial.println("IFTTT trigger failed!");
    }
    http.end();
    
    
  }
  else if(lastState == LOW && currentState == HIGH)
  {
    //released
  }
  lastState = currentState;
}

Compile and download the code to ESP32 board. Some points may need attention: 1, make sure the board is “FireBeetle-ESP32” 2, always select the right serial port, 3, please change the upload speed Frequency to lower value if multiple flashing failures are observed. Eg make the upload speed to “460800”.

Figure 13. adjust arduino flashing settings

Try the program, press the button, and wait in the mail box.

Bingo, after few seconds, we can find a new mail from IFTTT, it works!
