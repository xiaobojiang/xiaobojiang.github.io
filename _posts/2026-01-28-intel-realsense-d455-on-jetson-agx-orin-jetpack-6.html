This document summarizes the complete, working procedure to use Intel RealSense D455 on Jetson AGX Orin (JetPack 6.x) with librealsense, RSUSB backend, and Python (pyrealsense2).
It also documents why each step is required, based on real failure modes encountered.

1. Background / Key Constraints

On JetPack 6.x, the default kernel + V4L2 stack is incompatible with RealSense D4xx devices for full sensor enumeration.

Symptoms:

  • lsusb sees the camera

  • v4l2-ctl shows multiple /dev/video*

  • guvcviewer may show partial streams

  • realsense-viewer → No device found

Root causes

  • Missing kernel features required by librealsense

  • V4L2 backend fails on Jetson JP6

  • USB interface permission handling differs from x86

Solution overview

  • Build librealsense from source

  • Force RSUSB backend (libusb)

  • Add explicit USB interface udev rules

  • Build Python bindings from the same source

  • Use RSUSB-safe Python API pattern

2. System Requirements

  • Jetson AGX Orin

  • JetPack 6.x

  • Python 3.10

  • Intel RealSense D455

  • USB 3.x cable (direct connection, no passive hubs)

3. Remove Conflicting Packages

Remove any preinstalled RealSense packages:

sudo apt purge librealsense2* python3-pyrealsense2* -y
pip3 uninstall pyrealsense2 -y
rm -rf ~/.local/lib/python3.10/site-packages/pyrealsense2*

4. Build librealsense with RSUSB + Python

4.1 Install dependencies

sudo apt install -y \
  git cmake build-essential libusb-1.0-0-dev \
  libgtk-3-dev libssl-dev pkg-config

4.2 Get source (recommended version)

git clone --depth=1 --branch v2.55.1 https://github.com/IntelRealSense/librealsense.git
cd librealsense

4.3 Configure build (CRITICAL FLAGS)

mkdir build && cd build

cmake .. \
  -DFORCE_RSUSB_BACKEND=ON \
  -DBUILD_GRAPHICAL_EXAMPLES=ON \
  -DBUILD_PYTHON_BINDINGS=ON \
  -DPYTHON_EXECUTABLE=/usr/bin/python3 \
  -DPYTHON_LIBRARY=/usr/lib/aarch64-linux-gnu/libpython3.10.so \
  -DPYTHON_INCLUDE_DIR=/usr/include/python3.10 \
  -DCMAKE_BUILD_TYPE=Release

4.4 Build & install

make -j$(nproc)
sudo make install
sudo ldconfig

This installs:

  • librealsense2.so → /usr/local/lib

  • realsense-viewer

  • pyrealsense2 → /usr/lib/python3/dist-packages

5. USB Permission Fix (MANDATORY on Jetson)

5.1 Why default rules fail

Intel’s default udev rules:

  • Match only USB devices, not USB interfaces

  • Work on x86

  • Fail on JetPack 6 with RSUSB

Result:

RS2_USB_STATUS_ACCESS
failed to set power state

5.2 Required udev rule (critical fix)

Create a new rule file:

echo 'SUBSYSTEM=="usb", ATTR{idVendor}=="8086", MODE="0666"' | \
  sudo tee /etc/udev/rules.d/98-realsense-usb.rules

Reload and apply:

sudo udevadm control --reload-rules
sudo udevadm trigger

Unplug and replug the camera.

6. Verify Installation

6.1 Viewer

realsense-viewer

Expected: D455 appears and streams.

6.2 Python binding verification

python3 -c "import pyrealsense2 as rs; print(rs.__file__)"

Expected:

/usr/lib/python3/dist-packages/pyrealsense2/pyrealsense2.cpython-310-aarch64-linux-gnu.so

Check linkage:

ldd /usr/lib/python3/dist-packages/pyrealsense2/pyrealsense2*.so | grep realsense

Expected:

librealsense2.so => /usr/local/lib/librealsense2.so

7. Python Usage Rules (VERY IMPORTANT)

7.1 What NOT to use on Jetson + RSUSB

❌ Do NOT use:

config.resolve()
rs.pipeline_wrapper()

These APIs assume V4L2 backend and will fail with:

RuntimeError: No device connected

7.2 Correct Python pattern (RSUSB-safe)

import pyrealsense2 as rs
import numpy as np
import cv2

pipeline = rs.pipeline()
config = rs.config()

config.enable_stream(rs.stream.depth, 640, 480, rs.format.z16, 30)
config.enable_stream(rs.stream.color, 640, 480, rs.format.bgr8, 30)

profile = pipeline.start(config)

device = profile.get_device()
print("Connected device:", device.get_info(rs.camera_info.name))

try:
    while True:
        frames = pipeline.wait_for_frames()
        color_frame = frames.get_color_frame()
        if not color_frame:
            continue
        img = np.asanyarray(color_frame.get_data())
        cv2.imshow("Color", img)
        if cv2.waitKey(1) == 27:
            break
finally:
    pipeline.stop()

log disable of uvcdynctrl, otherwise the log (/var/log/uvcdynctrl-udev.log) exploded!

sudo nano /lib/udev/uvcdynctrl
Find debug=1 change to debug=0
Restart service
sudo udevadm control --reload
sudo udevadm trigger

On Jetson AGX Orin (JetPack 6), RealSense works reliably only with RSUSB + source-built Python bindings and RSUSB-safe API usage.

Once set up correctly, the system is stable and production-ready.
