We can use port forward to run remote service locally for debugging purpose. Sometimes we do not want to make ssh terminal visible.

Below command can help.
The ssh command with the flags -NfL localhost:9999:localhost:9999 your_user_name@remote_ip_address does the following:

-N: Do not execute a remote command. This is used to forward ports only.
-f: Requests ssh to go to background just before command execution. Be careful on this option, it really goes to background. we can check the PID via command pgrep -f "ssh command. eg, ssh -NfL 8000:127.0.0.1:8000 hp", and kill it.
-L: localhost:9999:localhost:9999: Specifies that the local port 9999 on your machine
should be forwarded to port 9999 on the remote machine (localhost).

This means that any traffic sent to port 9999 on your machine will be forwarded to port 9999 on the remote machine.

your_user_name@remote_ip_address: Specifies the username (your_user_name) and the IP address or hostname (remote_ip_address) of the remote machine.

Overall, this command establishes an SSH connection to the remote machine (remote_ip_address) and sets up port forwarding from port 9999 on your local machine to port 9999 on the remote machine, without executing any remote command (-N) and running in the background (-f).

Finally, recommand to use ssh -NL flags, which runs in foreground, and easy to kill if applicable.
