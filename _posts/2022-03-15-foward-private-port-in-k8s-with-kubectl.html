In common case, external traffic needs to go through ingress into the K8S cluster. Interal resources are not accessible to the outside. For debug or diagnostic purpose, we want to access internal port, eg, redis inside the cluster. Similar to ssh forward, K8S also provide a powerful command "kubectl port-forward" for this purpose.

the command looks like in this format:

kubectl port-forward <resource-type/resource-name> -n <namespace> [--address 0.0.0.0] [local_port]:<remote_port>

Here,

  • <resource-type/resource-name> is the resource and its unique name that listens the port in our k8s cluster, resource can be pod, service
  • [--address 0.0.0.0] is optional parameter, with this, means this local port listens traffic from all ip. default is empty, means only traffic from localhost is accepted.
  • <namespace> is the namespace where the resources are located
  • [local_port] is the local port that this forwarded service listened, it is optional, empty means it will be forwared to a random local port
  • <remote_pod> is the remote resource's port we want to forward to local

You can run port-forward on different resource in your cluster, eg, pod, deployment, services.

Sample command to perform port forwarding on pod: 

kubectl port-forward pods/my-pod 8080:80

Sample command to perform port forwarding on deployment: 

kubectl port-forward deployment/my-deployment 8080:80

Sample command to perform port forwarding on replicaset: 

kubectl port-forward replicaset/my-replicaset 8080:80

Sample command to perform port forwarding on service: 

kubectl port-forward service/my-service 8080:80

The command is actively running to keep this particular tunnel alive. we can press Ctrl + C to immediately end port forwarding.
