NVIDIA NIM (NeMo Inference Microservices) is NVIDIA’s containerized solution for running foundation models anywhere — from the cloud to your own GPU server. Each NIM is a pre-built Docker image that bundles a model and an optimized inference stack so you can start generating responses within minutes.

🧠 Why NIM?

✨ Highlights:

  • Plug-and-play deployment — one container = one model endpoint.

  • Optimized performance using TensorRT-LLM / vLLM for low-latency inference.

  • Standard APIs (OpenAI-compatible) for easy integration.

  • Flexible deployment — run in data centers, clouds, or at the edge.

  • Enterprise-grade scaling & observability with NVIDIA AI Enterprise and Kubernetes support.

In short: production-ready AI models, your infrastructure.

Install Nvidia container toolkit:

follow this guide  if this is not available on the machine: https://docs.nvidia.com/datacenter/cloud-native/container-toolkit/latest/install-guide.html

🔐 Setup — Access & Keys

  1. Create / log in to your NVIDIA NGC account.

  2. Accept the model license (e.g., Meta Llama 3 8B Instruct NIM) on its NGC page.

  3. Generate an API key under Setup → API Keys in your NGC account.

  4. Authenticate Docker to NVIDIA’s registry:

docker login nvcr.io

#username: $oauthtoken

#password: <your NGC API key>

(or use)

echo <YOUR_NGC_API_KEY> | docker login nvcr.io --username '$oauthtoken' --password-stdin

💻 Run Llama 3 NIM Locally

Create a volume

docker volume create nim-cache

Once logged in, simply pull and start the container:

export NGC_API_KEY=<YOUR_NGC_API_KEY>

Then if you are facing code 451, add a proxy to bypass

  -e HTTP_PROXY=http://<proxy_ip:port> \

  -e HTTPS_PROXY=http://<proxy_ip:port> \

  -e NO_PROXY=localhost,127.0.0.1,<proxy_ip> \

Full command:

docker run --gpus all \

  -p 18000:8000 \

  -e NGC_API_KEY \

  -e TRANSFORMERS_CACHE=/opt/nim/.cache \

  -v nim-cache:/opt/nim/.cache \

  -e HTTP_PROXY=http://<proxy_ip:port> \

  -e HTTPS_PROXY=http://<proxy_ip:port> \

  -e NO_PROXY=localhost,127.0.0.1,<proxy_ip> \

  nvcr.io/nim/google/gemma-3-1b-it:latest

✅ Tip:

If your machine has multiple GPUs, you can limit usage to one by adding:

--gpus '"device=0"'

The NIM container launches a REST API server on port 8000 — ready to receive inference requests.

If you are facing issues on GPU & cuda version capability and compatiblity "You can use float16 instead by explicitly setting thedtype flag in CLI, for example: --dtype=half."

Try below command:

python3 -m vllm_nvext.entrypoints.openai.api_server --dtype half

🧪 Quick Test — Call the Model API

Use openai lib to talk to your local endpoint:

from openai import OpenAI
#Point the OpenAI client to your local Ollama endpoint
client = OpenAI(base_url="http://localhost:18000/v1", api_key="not-needed")
#Send a chat completion request
response = client.chat.completions.create(
model="google/gemma-3-1b-it",
messages=[
{"role": "system", "content": "You are a helpful assistant."},
{"role": "user", "content": "Explain quantum computing in simple terms."}
],
)

response:

"""Okay, let’s break down quantum computing in a way that’s hopefully easy to understand! It’s a really weird and fascinating field, but here’s the gist:

1. Regular Computers vs. Quantum Computers:

Regular Computers (like your laptop): They store information as bits…."""

Print the result

print(response.choices[0].message.content)

You’ll get a neat JSON response containing the model’s reply — just like using OpenAI’s API, but fully local and GPU-accelerated. ⚡
