Currently latest maven in Ubuntu apt repository is 3.6.3

However, in apache website, latest version is already 3.8.6. I got one request from team is to upgrade maven used in Jenkins to this latest version.

Install maven binary manually 

1, download package from apache website

$ wget https://dlcdn.apache.org/maven/maven-3/3.8.6/binaries/apache-maven-3.8.6-bin.zip

2, unzip 

$ unzip apache-maven-3.8.6-bin.zip

3, move the total folder to /opt

$ sudo mv apache-maven-3.8.6 /opt/maven

4, create two environmental parameter.

Recommendation from website is in /etc/profile.d/maven.sh

For me, it is not working, I need to source it again.

I put it in my zshrc: ~/.zshrc

Content is:

export MAVEN_HOME=/opt/maven

export PATH=\$PATH:\$MAVEN_HOME/bin

5, try it

$ mvn -v [11:44:48]
Apache Maven 3.8.6 (84538c9988a25aec085021c365c560670ad80f63)
Maven home: /opt/maven
Java version: 11.0.17, vendor: Ubuntu, runtime: /usr/lib/jvm/java-11-openjdk-amd64
Default locale: en_US, platform encoding: UTF-8
OS name: "linux", version: "5.4.0-131-generic", arch: "amd64", family: "unix"

6, use it in Jenkins

Weird enough, I tried to set the folder in Jenkins configurations/Global Tool Configuration/Maven

Set it to local folder /opt/maven

Running mvn -v failed, cannot find the mvn package

So I use the environmental setting in Jenkins, configurations/global configurations/Global properties

7, works now in the Jenkins with new version
