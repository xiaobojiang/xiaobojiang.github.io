In normal case, openssl generates certs of x509 version 1. In some application scenario, due to security consideration, x509 version 3 certs are required, with v3 extension inside.

Following is a typical requirement for self-signed ca and devices.

Authority Root and Intermediate Certificate

Example root certificate

X509v3 extensions:
    X509v3 Key Usage: critical
        Digital Signature, Certificate Sign, CRL Sign
    X509v3 Basic Constraints: critical
        CA:TRUE, pathlen:1
    X509v3 Subject Key Identifier:
        03:DE:50:35:56:D1:4C:BB:66:F0:A3:E2:1B:1B:C3:97:B2:3D:D1:55
    X509v3 Authority Key Identifier:
        03:DE:50:35:56:D1:4C:BB:66:F0:A3:E2:1B:1B:C3:97:B2:3D:D1:55

Device Certificate

Example certificate for device with globally unique identifier 

X509v3 extensions:
           X509v3 Key Usage: critical
               Digital Signature, Key Encipherment
           X509v3 Basic Constraints: critical
               CA:FALSE, pathlen:0
           X509v3 Extended Key Usage:
               TLS Web Client Authentication, TLS Web Server Authentication
           X509v3 Subject Key Identifier:
               89:A7:49:8A:6F:AB:E4:F1:83:40:4A:05:0F:9F:84:69:AE:FC:36:9D
           X509v3 Authority Key Identifier:
               C8:A6:D5:5B:D9:1B:26:3F:BE:2E:CB:E5:44:73:29:BA:23:71:F3:18

This requires certificates with x509 version 3 extension.

How to generate such kind of certs? Do not worry, follow the steps in below, it is not very complex:

CA

Prepare one config file in folder ca/openssl.cnf

[ req ]
distinguished_name = req_distinguished_name
policy = policy_match
x509_extensions = v3_ca
# For the CA policy
[ policy_match ]
countryName = optional
stateOrProvinceName = optional
organizationName = optional
organizationalUnitName = optional
commonName = supplied
emailAddress = optional
[ req_distinguished_name ]
countryName = Country Name (2 letter code)
countryName_default = CN
countryName_min = 2
countryName_max = 2
stateOrProvinceName = State or Province Name (full name) ## Print this me
ssage
stateOrProvinceName_default = Shanghai ## This is the default value
localityName = Locality Name (eg, city) ## Print this message
localityName_default = Shanghai ## This is the default value
0.organizationName = Organization Name (eg, company) ## Print this messa
ge
0.organizationName_default = Iotcolon ## This is the default value
organizationalUnitName = Organizational Unit Name (eg, section) ## Print thi
s message
organizationalUnitName_default = Iotcolon ## This is the default value
commonName = Common Name (eg, your name or your server hostname)
## Print this message
commonName_max = 64
emailAddress = Email Address ## Print this message
emailAddress_max = 64
[ v3_ca ]
subjectKeyIdentifier = hash
authorityKeyIdentifier = keyid:always,issuer
basicConstraints = critical,CA:true,pathlen:1
keyUsage = critical,digitalSignature,keyCertSign,cRLSign
nsComment = "OpenSSL Generated Certificate"

Pay attention to following section in the file

x509_extensions=v3_ca

specifies the section that defines extensions to add to a self-signed certificate, where v3_ca is the name of the section. See the following [v3_ca] description for information about the fields that the section can contain. This field is optional.

Gen ca private key:

$ openssl genrsa -out ca/cakey.pem 4096

Generate ca cert

$ openssl req -new -x509 -days 3650 -config ca/openssl.cnf -key ca/cakey.pem -out ca/cacert.pem

client side

Prepare config file at devices/openssl.cnf

[ req ]
default_bits        = 4096
distinguished_name  = req_distinguished_name
policy              = policy_match
x509_extensions     = user_crt
req_extensions      = v3_req

[ req_distinguished_name ]
countryName                     = Country Name (2 letter code)
countryName_default             = CN
countryName_min                 = 2
countryName_max                 = 2
stateOrProvinceName             = State or Province Name (full name) ## Print this message
stateOrProvinceName_default     = Shanghai ## This is the default value
localityName                    = Locality Name (eg, city) ## Print this message
localityName_default            = Shanghai ## This is the default value
0.organizationName              = Organization Name (eg, company) ## Print this message
0.organizationName_default      = Signify ## This is the default value
organizationalUnitName          = Organizational Unit Name (eg, section) ## Print this message
organizationalUnitName_default  = Brightsites ## This is the default value
commonName                      = Common Name (eg, your name or your server hostname) ## Print this message
commonName_default              = 1234
commonName_max                  = 64
emailAddress                    = Email Address ## Print this message
emailAddress_default            = brightsites@singify.com
emailAddress_max                = 64

[ user_crt ]
## nsCertType              = client, server, email
nsComment               = "OpenSSL Generated Certificate"
keyUsage                = critical,digitalSignature,keyEncipherment
extendedKeyUsage        = clientAuth ##serverAuth for server certs
subjectKeyIdentifier    = hash
authorityKeyIdentifier  = keyid,issuer:always
basicConstraints        = critical,CA:FALSE,pathlen:0


[ v3_req ]
basicConstraints        = critical,CA:FALSE,pathlen:0
keyUsage                = critical,digitalSignature,keyEncipherment
extendedKeyUsage        = clientAuth ##serverAuth for server certs

Here,

req_extensions=v3_req

specifies the section that defines extensions to add to a certificate request, where v3_req is the name of the section. See the following [v3_req] description for information about the fields that the section can contain. This field is optional.

Gen private key and csr:

$ openssl req -new -days 3650 -nodes -keyout devices/2.key -out devices/2.csr -config devices/openssl.cnf

Sign certificate: 

$ openssl x509 -req -days 3650 -in devices/2.csr -CA ca/cacert.pem -CAkey ca/cakey.pem -CAcreateserial -out devices/2.crt -extensions user_crt -extfile devices/openssl.cnf

Note: server side is similar, just make the keyUsage as serverAuth

Check the v3 extension

Ca side

$ openssl x509 -text -noout -in ca/cacert.pem | grep -A15 "X509v3 extensions"

Client side

$ openssl x509 -text -noout -in devices/1.crt | grep -A15 "X509v3 extensions"

Done.
