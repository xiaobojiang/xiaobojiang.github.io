Milvus is a very nice and powerful vector database, it is one of the key component in AI RAG (retrieval augmented genereation) pipeline.

Here, this post is about how to remove the dependency of minio, due to its annoying license: AGPL V3.0, by consulting legal team, it is strongly unapproved for use in commercial release.Create S3 with default setting. I think that should also be applied to other business scenarios.

Steps to replace:

A S3 bucket is required for milvus vector db deployment, milvus will store logs and index files in the S3.

Please find some reference here:

A bucket with default settings, and name, eg ”milvusobjectstore” shall be created.

Please select the same region as EC2, region will be used in below steps

Key parameters:

  • Object Ownership: ACLs disabled

  • Block all public access

  • Bucket version: disable (depends)

  • Default encryption: SSE-S3

  • Buket key: Enable

After it is created, please be noted that an access key and secret shall be created for milvus to access.

Step 1: create a policy in IAM:

It grants access to this bucket only

{
            "Version": "2012-10-17",
            "Statement": [{
                "Effect": "Allow",
                "Action": [
                    "s3:*"
                ],
                "Resource": [
                    "arn:aws:s3:::<bucket name here>",
                    "arn:aws:s3:::<bucket name here>/*"
                ]
            }]
}

 Step 2

Create an user for program access with this policy attached (console access disabled)

And copy the access key and access secret for later use.

Prepare below docker-compose.yaml.

Put it under host folder.

Start service: $docker-compose up -d

Docker-compose.yaml

version: '3.5'

services:
  etcd:
    container_name: milvus-etcd
    image: quay.io/coreos/etcd:v3.5.5
    restart: always
    environment:
      - ETCD_AUTO_COMPACTION_MODE=revision
      - ETCD_AUTO_COMPACTION_RETENTION=1000
      - ETCD_QUOTA_BACKEND_BYTES=4294967296
      - ETCD_SNAPSHOT_COUNT=50000
    volumes:
      - ${DOCKER_VOLUME_DIRECTORY:-.}/volumes/etcd:/etcd
    command: etcd -advertise-client-urls=http://127.0.0.1:2379 -listen-client-urls http://0.0.0.0:2379 --data-dir /etcd

  standalone:
    container_name: milvus-standalone
    image: milvusdb/milvus:v2.3.2
    command: ["milvus", "run", "standalone"]
    restart: always
    environment:
      ETCD_ENDPOINTS: etcd:2379
      MINIO_ADDRESS: "s3.{{ S3_AWS_REGION }}.amazonaws.com"
      MINIO_PORT: 80
      MINIO_ACCESS_KEY_ID: {{ S3_ACCESS_KEY_ID }}
      MINIO_SECRET_ACCESS_KEY: {{ S3_SECRET_ACCESS_KEY }}
      MINIO_BUCKET_NAME: {{ S3_BUCKET_NAME }}
      AWS_REGION: {{ S3_AWS_REGION }}
    volumes:
      - ${DOCKER_VOLUME_DIRECTORY:-.}/volumes/milvus:/var/lib/milvus
    ports:
      - "19530:19530"
      - "9091:9091"
    depends_on:
      - "etcd"

networks:
  default:
    name: milvus

Two containers shall be up and running:
