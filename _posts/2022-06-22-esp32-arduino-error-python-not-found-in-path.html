I encountered this error, when I installed ESP32 board manager in the Arduino (1.8.19) on the MacBook, and compling & upload project to the board.

It complains that python is not found in the path.

well, well, it is strange, I checked my $PATH, it includes parent path of Python from Brew, and my Python from pyenv. And I can type python in terminal to start.

Again, searched for it for long time, and I solved this.

The cause of issue:

ESP32 on arduino seems link only search for python2, which is not available on my mac.

How to fix it:

Just find the platform.txt of ESP32, and replace the "python" with "python3".

compile and upload again, now it shall work as expected.

$ sed -i -e 's/=python /=python3 /g' ~/Library/Arduino15/packages/esp32/hardware/esp32/*/platform.txt
