Kivy is a UI library of Python, it can help developer quickly build a GUI program. Basics of Kivy can be found my previous post here (Power tool to build UI with python: Kivy  https://www.iotcolon.com/?p=142)

I spend some time to make a simple serial debugger based on Kivy, it is published on my git (https://github.com/xiaobojiang/kivy_playground).

In following few posts, I would like to share some experience I learned during the development.

First post is about UI.

UI with Kivy Language

Following is the running UI of the serial debugger, the layout and elements are fully composed with Kivy language.

Figure 1. My debugger's UI

My method is from macro to micro, always start from the whole GUI, then split the GUI into few main blocks, and in each block, split the block with more detailed elements.

For the serial debugger, the upper level block is like below code. The Root has a box layout, and by default it is horizontal, and this Root box has two elements, both of them are also box layouts. Box1 consume 30% width, by specifying size_hint_x: 0.3, and Box2 consumes other 70%.

<Root>:
    BoxLayout:
        orientation: 'horizontal'
        spacing: dp(10)
        canvas.before:
            Color:
                rgba: (0.179, 0.191, 0.191, 1)
            Rectangle:
                size: self.size

        BoxLayout:  #Box1
            orientation: 'vertical'
            size_hint_x: 0.3

            .....
                


        BoxLayout: #Box 2
            orientation: 'vertical'
            ....

Figure 2. Main breakdown of the UI

Recursively, for the Box1, I split the box into three boxes: Serial setting box Box1-1, Recv setting box Box1-2 and send setting box Box1-2.

And in each box, labels and button are added. For example, in Box1-1, there is a header text: "Serial Setting", following by selection of port, baudrate, parity, stop bi, data bits and open button.

Figure 3. breakdown of Box1

Furthermore, in the Serial setting box's selection part, it has two rows, one row is description of various serial port setings, like “port”, and other row is a drop down selection widget. Here, the GridLayout is used to make the alignment happen. Drop down selection utilizes the Spinner widget from Kivy.

One interesting point is about the checkbox, eg “Append \r\n” in Box1-3. Since I set the background to dark one, this makes the default frame color of the checkbox totally un-identifiable from the background.  

Here, the checkbox has special designed canvas to tune the color. 

      CheckBox:
	size_hint_y: None
	height: '16dp'
	id: id_append_linend_send
	active: False
	on_active: root.on_append_linend_send_change(self.active)
	pos: self.parent.center_x - dp(16), self.parent.center_y - dp(16)
	canvas.before:
		Color:
			rgb: 1,1,0
		Rectangle:
			pos:self.center_x-dp(8), self.center_y-dp(8)
			size:[dp(16),dp(16)]
		Color:
			rgb: 0.179, 0.191, 0.191
		Rectangle:
			pos:self.center_x-dp(7), self.center_y-dp(7)
			size:[dp(14),dp(14)]

And the highlighted effect after light yellow customization:

Figure 4. check box with light yellow color

In the serial setting box, there is an indicator (red/green) to show the serial connection status. A widget is defined in the kv file. The Color attribute is controlled by python file to switch color when status changes.

<Indicator>:
    canvas:
        Color:
            rgba: self.color
        Ellipse: 
            pos: self.center_x / 2 , self.center_y - dp(8)
            size: dp(16), dp(16)

On the right Box2, things are more simple, there are two large text area for received data and send data area.

there is also a checkbox "add timestamp", user can use this to add datetime at head of each receiving line.

There are also two buttons to clear received and sent text.

In the Box 1’s encoding area, user can choose ASCII or HEX encoding of the received and sent text in Box2.

There is also on handy function in Box 1: “save to file”, it will save all the receiving text into a txt file.

“SaveDialog” widget in below is design, let user to choose destination folder and specify the file name.

Figure 5. save dialog window

<SaveDialog>:
    text_input: text_input
    BoxLayout:
        size: root.size
        pos: root.pos
        orientation: "vertical"
        FileChooserIconView:
            id: filechooser
            filters: ['*.txt']
            rootpath: root.homepath
            on_selection: text_input.text = self.selection[0] if self.selection else ''

        TextInput:
            id: text_input
            size_hint_y: None
            height: '30dp'
            multiline: False

        BoxLayout:
            size_hint_y: None
            height: '30dp'
            Button:
                text: "Cancel"
                on_release: root.cancel()

            Button:
                text: "Save"
                on_release: root.save(filechooser.path, text_input.text)

Finally, in order to make the UI layout works for different screen resolution, all the size, height, width, padding are in the unit dp (density-dependent pixels)

For some size input, which only takes pixels, a method dp() is used to do the convert.

Hooks with Python script

In order to make the widget, like button, hooked with python script, one unique id should be assign, and claimed those ids at root level. we will talk abou usage in next post. 

<Root>:
    console_text: console_text
    send_text: send_text
    id_commport: id_commport
    id_baudrate: id_baudrate
    id_checksum: id_checksum
    id_stopbit: id_stopbit
    id_databit: id_databit
    id_indicator: id_indicator
    id_switch_serial: id_switch_serial
    id_recv_encode: id_recv_encode
    id_send_encode: id_send_encode
    id_append_linend_send: id_append_linend_send
    id_add_timestamp: id_add_timestamp

For example, in below code, this spinner can enable user change baudrate, one id "id_baudrate" is assigned.

Furthermore, "on_text" also defines a callback function "root.on_baudrate_change", when Spinner's text changes. If user change the selection of baudrate, this "on_text" function will be called. 

                      Spinner:
                        id: id_baudrate #unique id assigned
                        text: '9600'
                        values: ('1200', '2400', '4800', '9600', '19200', '38400', '57600', '115200')
                        size_hint_y: None
                        height: '30dp'
                        on_text: root.on_baudrate_change(self.text) #callback func when text change
                        size_hint_x: 0.618
