When you are deleting the cloudformation stacks, if you encounter this error:

"Failed to delete stack: Role arn:aws:iam::role/XXXX is invalid or cannot be assumed"

This usually happens when a role required to delete the stack has been accidentally deleted. You may get the error message

Role arn:aws:iam::<account>:role/<role name> is invalid or cannot be assumed.

This error will persist and blocks you from deleting remaining stacks.

How to fix?

One way to do this, is to create another role with good enough permissions.

Go to IAM > roles > create role > click on cloudformation for the service > make sure you give it the right permissions so that cloudformation can delete the stack.

And in next page, select permissions with admin for my case.

You should now be able to delete the stack

With command:

$ aws cloudformation delete-stack --stack-name RdsStack --role-arn arn:aws:iam::<account id>:role/<you new role name>

Remember to delete this power role after stack is deleted.

Another way is to create the role with same name as in the error message, with this, we shall be able to delete it also in the web console.
