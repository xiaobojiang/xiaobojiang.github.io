Intranet penetration is very useful for me, for example, exposing a device to internet with a public IP to access, exposing a intranet nas for public access.

There are few public commercial hw/sw to do this.

FRP is a free and opensource one, and it is quite easy to use.

What you need

1, a server with public IP or domain name

2, firewall management access of this server

The steps are splitted into two steps: 1, on server side, 2, on client side.

Install on server

Go to https://github.com/fatedier/frp/releases

1 Download the right arch copy

go to release directory, and select package with same arch as your server side.

Eg: wget https://github.com/fatedier/frp/releases/download/v0.36.2/frp_0.36.2_linux_amd64.tar.gz

2 install 

$ tar -zxf frp_0.36.2_linux_amd64.tar.gz
$ mv  frp_0.36.2_linux_amd64 frp
$ cd frp

3, change the frps.ini

$ vim frps.ini 

[common]
bind_port = <bind port>
token = <password>
authentication_timeout = 0
vhost_http_port = 8080
vhost_https_port = 8443

Remember to enable tcp access of bind_port to all ips in the firewall.

4, start the server

$ ./frps -c frps.ini

5, persistence the service and autostart

write a script to make frp service at server side auto-start.

create a file:

vi /etc/systemd/system/frps.service

content is

[Unit]
Description=frps
After=network.target

[Service]
ExecStart=/root/frp/frps -c /root/frp/frps.ini

[Install]
WantedBy=multi-user.target

6, start the service 

systemctl start frps.service

check the service status

systemctl status frps.service

make service auto start

systemctl enable frps.service

Install on Client Side

for openwrt OS:

1, install, similar to server

https://github.com/kuoruan/openwrt-frp

download, and upload and 

opkg install frpc_0.33.0-2_mipsel_24kc.ipk

2, create a config file for client service

$ vim /etc/frp/config

Content is:

[common]
server_addr = <server ip or domain name>
server_port = 57000
token = <password>
[ssh]
type = tcp
local_ip = 127.0.0.1
local_port = 22
remote_port = 58222

[common] section shall fills in with the server side credentials and server ip+port.

On the next section, eg [ssh]

and the section name [ssh] cannot be duplicated in different client's config file.

type can be tcp or secured tcp, details can be found in the github site: https://github.com/fatedier/frp

local ip and port is the port that we want to expose on the internet. we can access <local_ip>:<local_port> via <server_addr>:<remort port>

also, make sure the remote_port is also opened in the firewall at server side.

2, auto start

$ vim /etc/init.d/frp

Content is:

$ vim /etc/init.d/frp
#!/bin/sh /etc/rc.common
# Copyright (C) 2006-2011 OpenWrt.org

START=99

start() {
sleep 60; nohup /usr/bin/frpc -c /etc/frp/config >/root/nohup.out 2>&1 &
}

stop() {
kill -9 `ps | grep '/usr/bin/frpc' | grep -v 'grep' | awk '{print $1}'`
}

execute following commands:

$chmod +x /etc/init.d/frp
$/etc/init.d/frp enable

For CentOs or Ubuntu OS:

steps are similar

1, download the frpc package from github, and unzip (steps are same as server side)

2, prepare a frpc config file (same as openwrt client)

3, auto start

create a service config file:

$vim /etc/systemd/system/frpc.service

Content is:

[Unit]
Description=frpc
After=network.target
[Service]
ExecStart=/home/root/frp/frpc -c /home/root/frp/frpc.ini
[Install]
WantedBy=multi-user.target

Execute following command:

$ systemctl start frpc.service
$ systemctl enable frpc.service

Done, now try your service on <server_addr>:<remote port>
