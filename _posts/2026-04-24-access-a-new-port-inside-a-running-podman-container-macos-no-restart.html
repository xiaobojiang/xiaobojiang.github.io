If your container (e.g. Jupyter) starts a service on a new port like 9999, you can still access it from your browser without restarting — even though Podman doesn’t let you add new port mappings after launch.

Trick: SSH Tunnel into the Podman VM

Podman on macOS runs containers inside a VM (podman machine), so you can forward traffic through that VM.

1. Get Container IP

podman inspect -f '{{range .NetworkSettings.Networks}}{{.IPAddress}}{{end}}' jupyter

Example output:

10.88.0.2

2. Get VM SSH Info

podman machine inspect

Look under SSHConfig for:

  • RemoteUsername

  • Port

  • IdentityPath

3. Create Tunnel

ssh -i <IdentityPath> \
  -p <Port> \
  -L 8000:10.88.0.2:8000 \
  <RemoteUsername>@127.0.0.1

4. Open in Browser

http://localhost:8000

you will get something from the server

Hello, World!

Why This Works

  • Container runs inside Podman VM

  • VM is reachable via SSH

  • SSH tunnel bridges your Mac → VM → container

When to Use

  • You forgot to publish a port (-p)

  • You started a new service inside container

  • You don’t want to restart anything

Pro Tip

For future runs, expose a port range:

podman run -p 8000-10000:8000-10000 ...

Clean, fast, and no container restart needed.
