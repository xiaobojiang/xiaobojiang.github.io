Kivy is a easy to use, and cross platform library, packaging in Macos is mentioned in previous post (https://www.iotcolon.com/?p=203).

In this post, let’s talk about packaging executable for Window platform.

First of all, using pip to install pyinstaller, this software package is the key to build.

$ pip install pyinstaller

Then let’s use the pyinstaller to generate one basic spec file for our program.

$ python -m pyInstaller --onefile -w --name apploger --icon ..\applelightbaker\DataLoggerApp\icon.png ..\applelightbaker\DataLoggerApp\main.py

Here, some instructions on the parameters:

  • --onefile: this will enfore pyinstaller to generate one exe file which contains all referenced libs and data files
  • -w (--windowed, --no-console): do not provide console window for standard io. Without this option, you will see one additional black console besides our kivy GUI window.
  • --name: target name of final executable
  • --icon: file path of desired application icon image.
  • <…main.py>: this is the path of target python script file we would like to build executable

After we run this command, we will get a file: apploger.spec

We need to make additional changes to make it run correctly.

1, bundling data files in Analysis

add data files, eg:

datas=[
    ('..\\applelightbaker\\DataLoggerApp\\icon.png','.'),
    ('..\\applelightbaker\\DataLoggerApp\\font.ttf','.')
],

Here, The datas is a list of tuple. Each tuple contains two elements: 1, the data file we would like to add in the executable (eg, icon.png will be used in the program as avatar) 2, the path, ‘.’ Means that the file is copied in the same folder as python script file, you can specify ‘db’ to copy the file to the folder ‘db’

2, For kivy packaging, let’s add the dependencies hooks to correctly build the exe. Open the spec file and add these lines at the beginning of the spec (assuming sdl2 is used, the default now):

from kivy_deps import sdl2, glew

3, add Tree, and exclude files

find COLLECT() and add the data for our ap loger: Change the line to add a Tree() object, e.g. Tree('examples-path \\'). This Tree will search and add every file found in the examples-path directory to your final package.

To add the dependencies, before the first keyword argument in COLLECT add a Tree object for every path of the dependencies. E.g. *[Tree(p) for p in (sdl2.dep_bins + glew.dep_bins)] 

This step is important, it copes other important files to the executable, eg, kv files. If this steps is missing, you will get a frustrating error: “SDL2 cannot load”.

Also maintain the excludes list to remove those virtual environment and other temp files, so that the final executable size is optimized.

It will look like:

exe = EXE(
    pyz,
    Tree('..\\DataLoggerApp\\',
        excludes=['venv','.idea','*.txt','log','saved_data','*.cfg']),
    a.scripts,
    a.binaries,
    a.zipfiles,
    a.datas,
    [],
    name='apploger',
    debug=False,
    bootloader_ignore_signals=False,
    strip=False,
    upx=True,
    *[Tree(p) for p in (sdl2.dep_bins + glew.dep_bins)],
    upx_exclude=[],
    runtime_tmpdir=None,
    console=False,
    disable_windowed_traceback=False,
    argv_emulation=False,
    target_arch=None,
    codesign_identity=None,
    entitlements_file=None,
    icon='..\\DataLoggerApp\\icon.png',
)

4, Cool, now we can build windows executable with this simple command

$ pyinstaller ./apploger.spec

In later phase, if we change the script file, we do not need to repeate step 1-3, just running step 4 is sufficient.

Special tips:

How to find the bundled data files by the windows executable in the runtime?

In the runtime, these bundled data files can be found by your program by os.path.dirname(__file__), it is a temp folder for one file build, normally it is in C:\User\you\AppData\Local\Temp\_MEIXXX folder

So you can find the icon file or font file by the path: os.path.dirname(__file__)

In the standard document, you can also do this by:

import os, sys
from kivy.resources import resource_add_path, resource_find
if __name__ == '__main__':
    if hasattr(sys, '_MEIPASS'):
        resource_add_path(os.path.join(sys._MEIPASS))
YourApp().run()


 

OK, with those steps, you can already build a nice and running windows executable program.

Have fun!
