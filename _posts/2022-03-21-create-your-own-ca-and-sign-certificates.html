In most scenarios, we do not need to create own CA and sign certificates.

We can apply certificate pair for our website from AWS, or from other cloud vendor, this will cost you bit. We can also apply free certificates with tools, eg "Let's Encrypt".

However, in certain circumstance, we need to create and sign certificate by ourself, typical usage like openvpn, MQTT connectivity.

Let's share a little bit on how:

Step 1, create CA

create a new folder, called ca.

$ mkdir ca

create ca's private key

$ openssl genrsa -out ca/ca.key 4096

create ca's certificate signing request file (csr), in this step, the prompt will require input, eg common name, city.etc

$ openssl req -new -key ca/ca.key -out ca/ca.csr

sign the ca's certificate with ca's private key, -days specifies the valid time period in days. 

$ openssl x509 -req -days 3650 -in ca/ca.csr -signkey ca/ca.key -out ca/ca.crt

Step 2, sign devices keys with ca

create a folder to host devices certs, steps are same for server.

$ mkdir devices

create the private key for the client

$ openssl genrsa -out devices/1.key 4096

create the certificate signing request file

$ openssl req -new -key devices/1.key -out devices/1.csr

use ca to sign the device certificates

$ openssl x509 -req -days 3650 -in devices/1.csr -CA ca/ca.crt -CAkey ca/ca.pem -out
devices/1.crt -CAcreateserial

Step 3: verify the certificates

after certificate creation, we can view the certificate in text via

it will show many information, eg, issuer, validity, subject, public key info.

$ openssl x509 -in devices/1.crt -text -noout

we can verify that a certificate was issued by a specific CA, given that CA's certificate.

$ openssl verify -verbose -CAfile ca/ca.crt devices/1.crt                                                                                       
devices/1.crt: OK

That's it, a simplest way to create your own ca and sign your certificates.
