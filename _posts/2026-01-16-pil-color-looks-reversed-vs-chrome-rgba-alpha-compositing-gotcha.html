When composing diffusion model outputs using PIL, images may appear to have reversed, dark, or incorrect colors compared to how the same PNG looks in Chrome / browsers.

This is not a model issue.

It is caused by incorrect handling of RGBA alpha channels when converting images to RGB.

Symptoms

  • Output PNG looks correct in Chrome

  • Same image looks:

    • darker

    • washed out

    • “color inverted” or wrong

  • Issue appears after:

    • Image.open(...).convert("RGB")

    • PIL image composition / tiling / saving

Root Cause (Confirmed)

PIL drops alpha incorrectly by default

Many diffusion pipelines (including Qwen Image Layered) output images in RGBA mode.

When this is done:

img = Image.open("output.png").convert("RGB")

PIL:

  • discards the alpha channel

  • implicitly composites the image over a black background

Browsers (Chrome, Safari, etc):

  • correctly composite RGBA images over a white background

Result

This difference explains the mismatch.

How to Confirm

from PIL import Image

img = Image.open("0.png")

print(img.mode)

If output is:

RGBA

You are affected.

❌ Incorrect (Common Mistake)

img = Image.open(path).convert("RGB")

This silently breaks RGBA images.

✅ Correct Fix: Explicit Alpha Compositing

Always composite RGBA images onto a known background before converting.

Safe utility function

from PIL import Image

def rgba_to_rgb(img, bg_color=(255, 255, 255)):

    if img.mode == "RGBA":

        bg = Image.new("RGB", img.size, bg_color)

        bg.paste(img, mask=img.split()[-1]) # use alpha channel

        return bg

    return img.convert("RGB")

Use it everywhere you load images

img = rgba_to_rgb(Image.open(path))

Best Practices

✅ Always check image mode

✅ Explicitly composite RGBA → RGB

❌ Never call .convert("RGB") directly on RGBA diffusion outputs

✅ Choose a known background (white, checkerboard, etc.)

Optional: Preserve Transparency Instead

If you want to keep alpha (e.g. for UI usage):

img = Image.open(path).convert("RGBA")

Only flatten to RGB at the final export step.
