On the aws console, from last year, there is a new feature that you can view resources of EKS cluster in the console.

However, I find I am blocked from viewing them.

It complains: "Your current user or role does not have access to Kubernetes objects on this EKS cluster This may be due to the current user or role not having Kubernetes RBAC permissions to describe cluster resources or not having an entry in the cluster’s auth config map.Learn more"

Here is the quick solution:

When you create an Amazon EKS cluster, the AWS Identity and Access Management (IAM) entity user or role, such as a federated user that creates the cluster, is automatically granted system:masters permissions in the cluster's role-based access control (RBAC) configuration in the Amazon EKS control plane.

-> To grant additional AWS users or roles the ability to interact with your cluster, you must edit the aws-auth ConfigMap within Kubernetes of that EKS cluster. This has to be done assuming the cluster creator (role/user) as only the cluster creator has the admin access over it. To know more information please refer the document [1].

-> If you are the cluster creator you can directly edit the aws-auth ConfigMap and if you are not the cluster creator you need to assume the cluster creator and edit the aws-auth configmap of the EKS cluster using the command below. $ kubectl edit configmap aws-auth -n kube-system

Add the IAM user to mapUsers.

For example: 

mapUsers: |
- userarn: arn:aws:iam::XXXXXXXXXXXX:user/testuser
username: testuser
groups:
- system:masters

Or add the IAM role to mapRoles.

For example: 

mapRoles: |
- rolearn: arn:aws:iam::XXXXXXXXXXXX:role/testrole
username: testrole
groups:
- system:masters

-> The detailed process is mentioned in [2] and I recommend you to go through it and perform those steps to resolve the issue.

-> After performing these steps you should be able to access the EKS cluster resources.

Please note If you are giving system:masters under groups, The system:masters group allows superuser access to perform any action on any resource. Also make sure you are editing the aws-auth ConfigMap for appropriate cluster.

After add my user into the config, it works fine:

[1] https://docs.aws.amazon.com/eks/latest/userguide/add-user-role.html

[2] https://aws.amazon.com/premiumsupport/knowledge-center/eks-api-server-unauthorized-error/

[3] https://docs.aws.amazon.com/eks/latest/userguide/view-kubernetes-resources.html
