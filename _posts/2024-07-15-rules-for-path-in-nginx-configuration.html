In NGINX, path rules are used to determine how to handle incoming requests based on the URI (Uniform Resource Identifier). The ~ and ~* operators are used to define regular expression (regex) matching rules, while = and ^~ are used for literal string matching and prefix matching respectively. Here's an explanation of each:

1. =

The = operator is used for exact string matching. If the URI exactly matches the specified string, NGINX will use this rule.

Example:

location = /exact-match 
{
 # Configuration for exact match 
}

In this case, only requests to /exact-match will match this location block.

2. ^~

The ^~ operator is used for prefix matching. If a URI starts with the specified prefix, NGINX will use this rule and stop searching for other matches.

Example:

location ^~ /images/ {    # Configuration for URIs starting with /images/ }

In this case, any request starting with /images/ will match this location block.

3. ~

The ~ operator is used for case-sensitive regular expression matching.

Example:

location ~ \.jpg$ 
{
 # Configuration for URIs ending with .jpg 
}

In this case, any URI ending with .jpg will match this location block, and the match is case-sensitive (e.g., /picture.JPG would not match).

4. ~*

The ~* operator is used for case-insensitive regular expression matching.

Example:

location ~* \.jpg$ 
{
 # Configuration for URIs ending with .jpg 
}

In this case, any URI ending with .jpg will match this location block, and the match is case-insensitive (e.g., /picture.JPG would match).

Matching Order

NGINX processes the location blocks in a specific order:

  1. Exact matches (=).

  2. Prefix matches (^~).

  3. Case-sensitive regex matches (~).

  4. Case-insensitive regex matches (~*).

  5. Prefix matches (default).

Example Configuration

Here's an example configuration combining these different rules:

server 
{ 
listen 80; 
location = /{
 # Configuration for the exact match of "/" 
} 
location ^~ /static/ {
 # Configuration for any URI starting with "/static/" 
}
 location ~ \.php$ {
 # Configuration for case-sensitive regex match of ".php" 
}
 location ~* \.jpg$ {
 # Configuration for case-insensitive regex match of ".jpg" 
}
 location / {
 # Default configuration for any other URI
 } 
}

In this configuration:

  • The exact match for / will be processed first.

  • URIs starting with /static/ will be processed next.

  • URIs matching the case-sensitive regex \.php$ will be processed after.

  • URIs matching the case-insensitive regex \.jpg$ will be processed next.

  • Any other URIs will be processed by the default location block.
