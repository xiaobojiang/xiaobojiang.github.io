Preparation

I encountered  issue of wget Zulu repository key.

So I wget locally, and copy inside

wget --quiethttps://cdn.azul.com/public_keys/alpine-signing@azul.com-5d5dc44c.rsa.pub

Someone mentioned wget issue in buildx may link to MTU size (reduce the size)

So I addcat /etc/docker/daemon.json

{

"mtu":1300

}

I think it is optional, try it if you face apk download issue.

Docker file 

FROM alpine:3.15

ENV LANG en_US.UTF-8

ENV LANGUAGE en_US:en

ENV LC_ALL en_US.UTF-8

COPYalpine-signing@azul.com-5d5dc44c.rsa.pub/etc/apk/keys

RUN apk --repositoryhttps://repos.azul.com/zulu/alpine--no-cache add zulu17-ca-jdk-headless~=17.0.2-r1 tzdata

ENV JAVA_HOME=/usr/lib/jvm/zulu17-ca

Here, compared to the original docker build file in GitHub(https://github.com/zulu-openjdk/zulu-openjdk/tree/master/alpine/17.0.2-17.32.13):

I did not install the zulu17-jdk, because it brings both zulu-ca-jdk, zulu-ca-jdk-headless, also zulu-jdk, zulu-jdk-headless.

Which from runing and license view, is not necessary.

So only the zulu17-ca-jdk-headless is installed, since UI is not required.

I use buildx to build multi-arch images

Build script:

docker buildx build -o type=registry --platform linux/arm64/v8,linux/amd64 --tagregistry.cn-shanghai.aliyuncs.com/sigcbp/zulujdkheadless:17.0

Done
