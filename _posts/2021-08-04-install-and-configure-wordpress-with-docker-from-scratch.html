Wordpress is the tool and currently I am using to build this site.

Prerequisite:

- domain name.
You can purchase one from Godaddy.com, or apply a free one from freenom.com
- cloud server instance.
I purchased on Centos 8 instance from Vultr, 1 core 2G is minimum to start with.

What you will go through and hands on during the journey:

1, set up Nginx and Let’s encrypt on the host machine

2, set up docker 

3, set up mysql and wordpress in docker env

4, config nginx and wordpress

Let’s dive into each step with more details


Nginx and Let’s encrypt

Nginx is a very nice reverse proxy, let’s encrypt provides free TLS certificates.

Nginx and Let’s encrypt works together to provide secure https access to your site.
When I start to play with, I did not plan to go all for docker, thus, nginx and let’s encrypt are installed on the host machine.

It is not that hard to pack them into a customized docker image.
First, let’s install these packages: 

#install nginx 
$ yum install nginx -y 
$ systemctl start nginx
$ systemctl enable nginx
#install certbot 
$ yum install certbot -y


Create https certificates 

$certbot certonly —nginx

Follow the cli guide, fill in the domain name(s), and it will create certificates in /etc/letsencrypt/live/iotcolon.com.

iotcolon.com is my domain name.

In this folder,  you will find the fullchain.pem and privacy.pem.

These files will be used for later use.

The certificates from Let’s encrypt have a valid period of 3 months.

We need a cronjob to regularly refresh https certificates
Add a script in root folder, named it as renew_certbot.sh

#!/bin/sh
certbot renew --pre-hook "service nginx stop" --post-hook "service nginx start"


add cron job

$crontab -e 
#in the pop, add following line
55 6,18 * * * /root/renew_certbot.sh


setup docker

Later, I found it is better to host wordpress and mysql in docker instead of host machine. This can bring a lot of benefits.
Steps to install docker on Centos 8/RHEL 8

1, remove the podman

podman is the replacement of docker in  Centos 8/RHEL 8, theoretically it is compatible with docker command, and provide non-root runtime. 

However, docker is the most familiar tool I used for years. 

You cannot directly install docker, this will conflict with podman, use below commands to remove it first.

2, install docker

detail shell comands as below:

#remove podman 
$yum remove podman buildah -y
#install docker
$ yum-config-manager --add-repo https://download.docker.com/linux/centos/docker-ce.repo 
$ yum remove podman buildah -y$ yum install -y yum-utils device-mapper-persistent-data lvm2
$ yum -y install docker-ce
#start docker and make it autostart
$ systemctl start docker
$ systemctl enable docker

deploy mysql and wordpress in docker

Please find the example code 
Start mysql export 

#!/bin/sh

export MYSQL_PASSWORD=<password>

if [ $( docker ps -a | grep mysql | wc -l ) -gt 0 ]; then
	docker stop mysql | xargs docker rm ;
fi

docker run -p 3306:3306 --name mysql --restart=always -v ~/data:/var/lib/mysql -e MYSQL_ROOT_PASSWORD=${MYSQL_PASSWORD} -e MYSQL_DATABASE=wordpress -d mysql:5.7 --max_allowed_packet=64M --max-connections=1000 --sql-mode=""


Start wordpress 

#!/bin/sh
export MYSQL_PASSWORD=<password>
export MYSQL_HOST=<mysql host>


if [ $( docker ps -a | grep wordpress | wc -l ) -gt 0 ]; then
	docker stop wordpress | xargs docker rm ; 
fi

docker run --restart=always  -e WORDPRESS_DB_USER=root -e WORDPRESS_DB_PASSWORD=${MYSQL_PASSWORD} -e WORDPRESS_DB_HOST=${MYSQL_HOST} \
	-e WORDPRESS_CONFIG_EXTRA="define('WP_HOME', 'https://www.iotcolon.com'); define('WP_SITEURL', 'https://www.iotcolon.com');" \
	--name wordpress -m 512m  -p 8080:80 -v "/root/html":/var/www/html -d wordpress


Note: set WP_HOME and WP_SITEURL with your desired https address, this can avoid wordpress redirect to http or localhost


config nginx and wordpress

In the nginx config file, specify the domain name you want to use.

And add sections of ssl, which uses the previous generate https keys.    

the nginx conf file location: /etc/nginx/nginx.conf

listen [::]:443 ssl ipv6only=on; # managed by Certbot
listen 443 ssl; # managed by Certbot
ssl_certificate /etc/letsencrypt/live/iotcolon.com/fullchain.pem; # managed by Certbot
ssl_certificate_key /etc/letsencrypt/live/iotcolon.com/privkey.pem; # managed by Certbot
include /etc/letsencrypt/options-ssl-nginx.conf; # managed by Certbot
ssl_dhparam /etc/letsencrypt/ssl-dhparams.pem; # managed by Certbot
