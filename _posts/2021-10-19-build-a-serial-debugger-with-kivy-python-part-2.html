This is the second post about building an app with Kivy: serial debugger.

In linked post (https://www.iotcolon.com/?p=181) , the serial debugger’s UI design was discussed.

Figure 1. Serial Debugger UI

This post talks the python script, which will work together with the kv file, to make the app really interact with user and device.

You can name the python script with whatever you like, just put the py file in the same folder with kv file.

Few classes need to be defined in the file:

  • Indicator: this class links to the Indicator widget in kv file, script can control its color via updating the color property in the class
  • SaveDialog: this class links to the pop up dialog of saving log, script can get the file path from its instance
  • Root: this class links to the Root widget in the kv file, the main logic interacts with UI happens here
  • SerialDebuggerApp: this class must be name with kv file name + “App”, by this, Kivy knows how to find the correct kv file. Here, the root widget will be created and run. This class will be called in the “__main__”.
  • SerialPort: this is a abstract layer to deal with serials, it is not directly UI related.

Everytime, a event happened on the UI, a hook defined in the kv file will be triggered. Take the log save button as example.

Figure 2. Button's action hook with python script

In the button’s kv language definition, the button click release will trigger its root parent widget’s show_save function. This function shall be defined in the Root widget class, and in the function, it creates a pop up save dialog, configure properties of the pop up, and open it in the front UI.

Other hook can be the text change in the drop-down selection, take one of the serial setting, “port” as example.

Figure 3: Spinner's text change hook to and text update by python script

The port can be chosen by user, once user make the selection, the on_text in the kv file will trigger the function on_commport_change in the Root class. In my implementation, it will update the value in the SerialPort class instance.

The ports shall be dynamically refreshed if any serial devices are connected or disconnected. Function of update_comm_ports and update port will do this refresh job. ‘update_comm_ports’ will list out all the active serial ports in the host computer, put them into a list, ‘update_port’ will use this list to update the values of “id_commport” drop-down.

In the ‘SerialDebuggerApp’ class, with Clock’s schedule_interval feature, this function will be called every 1 second:

Clock.schedule_interval(root.update_port, 1.0)

I will leave other callbacks and hooks to readers, you can also find some of helper functions I wrote for Serial communication, eg encoding switch between hex and ascii.

And finally, you can find a function flow chart.

Figure 4. Flowchar of serial debugger APP

In the beginning, app start two fixed interval task, one is the 1 second frequency serial port refreshing, the other one is the 1/60 second serial receiving check function.

In order to break the line of serial input, 100ms idle period is implemented. If there is no input bytes detected in serial receiving more than this period, it takes this as a complete line, and append \n at this line in the receiving area.

The hex and acsii encoding switch is a handy function, it would be good to show the hex content if you see mess code in the receiving area, and also enable user to send arbitrary content to device.

That’s almost all for the python script part, you can find the complete code in my git: https://github.com/xiaobojiang/kivy_playground
