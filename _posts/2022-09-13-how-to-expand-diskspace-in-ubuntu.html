Inside the new provisioned linux server, sometimes, we will find the diskspace is not fully utilized.

Check below example, current disk space arrangement

The ubuntu—vg-ubuntu—lv only takes 50% of the disk /dev/sda3, which mounts to root directory

We would like to use whole /dev/sda3.

First Step, pvresize

pvresize is a tool to resize Physical Volume which may already be in a volume group and have active logical volumes allocated on it.

gateway@gateway:~$ sudo pvresize -v /dev/sda3
Creating directory "/etc/lvm/archive"
Archiving volume group "ubuntu-vg" metadata (seqno 2).
Resizing volume "/dev/sda3" to 118644736 sectors.
No change to size of physical volume /dev/sda3.
Updating physical volume "/dev/sda3"
Creating directory "/etc/lvm/backup"
Creating volume group backup "/etc/lvm/backup/ubuntu-vg" (seqno 3).
Physical volume "/dev/sda3" changed
1 physical volume(s) resized or updated / 0 physical volume(s) not resized

Then lvresize

lvresize allows you to resize a logical volume.

-l, --extents [+|-]LogicalExtentsNumber[%{VG|LV|PVS|FREE|ORIGIN}]

$ sudo lvresize -r -l+100%FREE /dev/mapper/ubuntu--vg-ubuntu--lv
Size of logical volume ubuntu-vg/ubuntu-lv changed from <28.29 GiB (7241 extents) to 56.57 GiB (14482 extents).
Logical volume ubuntu-vg/ubuntu-lv successfully resized.
resize2fs 1.46.5 (30-Dec-2021)
Filesystem at /dev/mapper/ubuntu--vg-ubuntu--lv is mounted on /; on-line resizing required
old_desc_blocks = 4, new_desc_blocks = 8
The filesystem on /dev/mapper/ubuntu--vg-ubuntu--lv is now 14829568 (4k) blocks long.

Check the new disk space arrangement

All the available space is utilized.
