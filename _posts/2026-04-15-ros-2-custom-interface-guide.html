1. Overview

This guide walks through creating a separate ROS 2 interface package, managing it as a Git submodule, and consuming custom message and service definitions in a Python node.

whole workpackage

ros_ws/
└── src/
    ├── navigation_interfaces/   # your custom interface package, this will use git submoduel add
    └── navigation_nodes/        # Python node package
---

2. Create an Interface Package as a standalone git

  1. Create a ROS 2 package with ament_cmake:

   ros2 pkg create --build-type ament_cmake --license Apache-2.0 navigation_interfaces

  1. Create a new Git repository:

   cd navigation_interfaces
   git init

  1. Create directories for your interface files:

   mkdir msg
   mkdir srv

  1. Add your .msg and .srv files:

  • msg/NavigationStatus.msg

  • srv/StartNavigation.srv

sample content for NavigationStatus.msg

# NavigationStatus.msg
# A summary of the robot's current navigation state

# Standard ROS2 timestamp and frame info
std_msgs/Header header

# Current goal (pose in map frame)
geometry_msgs/PoseStamped goal

# Current robot pose in map frame
geometry_msgs/PoseStamped current_pose

# Navigation path progress [0.0 -> 1.0]
float32 progress

# Is navigation currently active?
bool active

# Current state as a string (e.g., "IDLE", "MOVING", "ARRIVED", "FAILED")
string state

# Estimated time remaining in seconds (if available)
builtin_interfaces/Duration estimated_time_remaining

# Error code or diagnostic flags
int32 error_code
string error_message

sample content for StartNavigation.srv

StartNavigation.srv

# Request
geometry_msgs/PoseStamped goal
bool reset_odometry
---
# Response
bool success
string message

4. Add Submodule in Your Workspace

In your main ROS 2 workspace (e.g., ros_ws):

git submodule update --init --recursive

1. Python Node Using the Interfaces

1) Create a file start_navigation_server.py in a Python package
Create a file start_navigation_server.py in a Python package (like navigation_nodes or py_navigation):

#!/usr/bin/env python3

import rclpy
from rclpy.node import Node

from navigation_interfaces.srv import StartNavigation
from navigation_interfaces.msg import NavigationStatus

class StartNavigationServer(Node):
    def __init__(self):
        super().__init__('start_navigation_server')
        # Create a service with type StartNavigation and callback
        self.srv = self.create_service(
            StartNavigation,
            '/navigation/start',
            self.start_navigation_callback
        )

        # Simulate a status publisher so graph is complete
        self.status_pub = self.create_publisher(NavigationStatus, '/navigation/status', 10)

    def start_navigation_callback(self, request, response):
        self.get_logger().info(f"Received request to start navigation with pose: {request.goal}")
        # Always return success (fill fields in your response)
        response.success = True
        response.message = 'Navigation accepted!'

        # Optionally publish one status
        status = NavigationStatus()
        status.header.frame_id = 'map'
        status.active = True
        status.state = 'STARTED'
        self.status_pub.publish(status)

        return response


def main(args=None):
    rclpy.init(args=args)
    node = StartNavigationServer()
    rclpy.spin(node)
    rclpy.shutdown()


if __name__ == '__main__':
    main()

Then make it executable:

chmod +x start_navigation_server.py

Build and source your workspace before running.
2) start_navigation_client.py:

 #!/usr/bin/env python3

import rclpy
from rclpy.node import Node

from navigation_interfaces.srv import StartNavigation
from geometry_msgs.msg import PoseStamped
from std_msgs.msg import Header

class StartNavigationClient(Node):
    def __init__(self):
        super().__init__('start_navigation_client')
        self.client = self.create_client(StartNavigation, '/navigation/start')

        while not self.client.wait_for_service(timeout_sec=1.0):
            self.get_logger().info('Service not available, waiting...')

    def send_request(self):
        # Create request and fill fields
        req = StartNavigation.Request()
        req.goal = PoseStamped()
        req.goal.header.frame_id = 'map'
        req.goal.header.stamp = self.get_clock().now().to_msg()
        req.goal.pose.position.x = 1.0
        req.goal.pose.position.y = 2.0
        req.goal.pose.position.z = 0.0

        future = self.client.call_async(req)
        rclpy.spin_until_future_complete(self, future)

        response = future.result()
        self.get_logger().info(f"Response: success={response.success}, message='{response.message}'")


def main(args=None):
    rclpy.init(args=args)
    node = StartNavigationClient()
    node.send_request()
    node.destroy_node()
    rclpy.shutdown()


if __name__ == '__main__':
    main()

3) Optional NavigationStatus Publisher Example (Standalone) navigation_status_publisher.py

 #!/usr/bin/env python3

import rclpy
from rclpy.node import Node
from navigation_interfaces.msg import NavigationStatus
from std_msgs.msg import Header

class NavStatusPublisher(Node):
    def __init__(self):
        super().__init__('nav_status_publisher')
        self.pub = self.create_publisher(NavigationStatus, '/navigation/status', 10)
        self.timer = self.create_timer(1.0, self.publish_status)

    def publish_status(self):
        msg = NavigationStatus()
        msg.header.frame_id = 'map'
        msg.active = True
        msg.state = 'RUNNING'
        self.pub.publish(msg)
        self.get_logger().info('Published status update')

def main(args=None):
    rclpy.init(args=args)
    node = NavStatusPublisher()
    rclpy.spin(node)
    rclpy.shutdown()

if __name__ == '__main__':
    main()

6. Tips

  • Always source the install space before running

  • Use submodule tagging to manage versions

  • Your custom interfaces package can be reused across nodes

  • You don’t parse .msg files manually — Python generated classes handle all fields

7. Summary

You now have a modular ROS 2 interface package with:

  • Managed message/service definitions

  • Git submodule integration

  • Python consumption via generated classes
