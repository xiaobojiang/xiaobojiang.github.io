Hardcoded secret is a terrible thing. That means your whole system will be at danger if some one access your code base, and it makes it impossible to manage if your code will be deployed to different stage. 

One solution out, is to move password out of code, and be part of system provisioning, like environmental parameters, or yml, and mounted to the running time.

However, these part will normally be scripted, and also managed in another git, and once the password is git, it never leaves.

There is another good way is to keep those parameters in some key server, and application will fetch them when needed and authorized. In aws, there is such a feature called parameter store.

"Parameter Store, a capability of AWS Systems Manager, provides secure, hierarchical storage for configuration data management and secrets management. You can store data such as passwords, database strings, Amazon Machine Image (AMI) IDs, and license codes as parameter values. You can store values as plain text or encrypted data.

Parameter Store offers these benefits:

  • Avoid hardcode in git: Improve your security posture by separating your data from your code.

  • Use a secure, scalable, hosted secrets management service with no servers to manage.

  • Store configuration data and encrypted strings in hierarchies and track versions."

User can create such parameters with name, value and select type (secure or plain text) in the parameter store. And then this parameter can be read and used.

Let’s first create a parameter store policy, and attach to the EKS fargate’s service account.

Exmaple policy is like, this policy only grant read access, change it if not applicable. (Below is CDK python code)

param_store_policy_document_json = {
                "Version": "2012-10-17",
                "Statement": [
                    {
                        "Effect": "Deny",
                        "Action": [
                            "ssm:PutParameter",
                            "ssm:DeleteParameter",
                            "ssm:DeleteParameters",
                            "ssm:DescribeParameters"
                        ],
                        "Resource": "*"
                    },
                    {
                        "Effect": "Allow",
                        "Action": [
                            "ssm:GetParameterHistory",
                            "ssm:GetParametersByPath",
                            "ssm:GetParameters",
                            "ssm:GetParameter",
                        ],
                        "Resource": "arn:aws:ssm:"+Aws.REGION+":"+Aws.ACCOUNT_ID+":parameter/testps-*"
                    }
                ]
            }


Attach to service account:

param_store_policy_document = iam.PolicyDocument.from_json(param_store_policy_document_json)

            param_store_policy = iam.Policy(self,
                'paramStorePolicy',
                policy_name='paramStorePolicy',
                document=param_store_policy_document
            )

            #use service account way, , you need to specify sa in pod yaml, eg: serviceAccountName: serviceaccount-paramstore-brightsite
            for namespace in namespaces:
                #create service account
                eksParamStoreServiceAccount = self.ekscluster.add_service_account(
                    'serviceaccount-paramstore-'+namespace,
                    name='serviceaccount-paramstore-'+namespace,
                    namespace=namespace
                )
                eksParamStoreServiceAccount.role.attach_inline_policy(param_store_policy)

Now, fire up a pod with this serviceaccount.

apiVersion: apps/v1
kind: Deployment
metadata:
  name: python-blue
  namespace: xiaobo
spec:
  replicas: 1
  selector:
    matchLabels:
      app: python
      version: blue
  template:
    metadata:
      name: python
      labels:
        app: python
        version: blue
    spec:
      serviceAccountName: serviceaccount-paramstore-xiaobo
      containers:
      - name: python
        image: python:3.9-slim-buster
        command:
        - sh
        - -c
        - while true; do sleep 3600; done

Install boto3 (https://boto3.amazonaws.com/v1/documentation/api/latest/index.html) :

$pip install boto3

Now try to read the parameter in plaintext or secureString way.

Plain text reading:

import boto3
AWS_REGION = "eu-central-1"
ssm_client = boto3.client('ssm',region_name=AWS_REGION)
get_response = ssm_client.get_parameter(Name='xiaobo-test',
                                        WithDecryption=False)
print(get_response)

Secure string parameter reading.

import boto3
AWS_REGION = "eu-central-1"
ssm_client = boto3.client('ssm',region_name=AWS_REGION)
get_response = ssm_client.get_parameter(Name='xiaobo-test-secure',
                                        WithDecryption=True)
print(get_response)

With parameter store, these parameters will be stored in the AWS cloud, and application can read them via name in a secure and managed way.
