When building a FastAPI application, it’s important to make sure critical resources like databases are properly initialized and cleaned up as part of your app’s lifecycle.

A clean and effective way to do this is by using Python’s @asynccontextmanager with FastAPI’s lifespan support.

🧠 What’s @asynccontextmanager and Why Use It?

The @asynccontextmanager decorator allows you to define asynchronous setup and teardown logic in one place, using yield to separate them.

It’s perfect for preparing things like databases, caches, or background tasks before the API starts handling requests.

✅ Example: Setting Up an SQLite Database Connection

Let’s walk through how to set up an SQLite database connection at app startup and close it cleanly at shutdown.

from fastapi import FastAPI
from contextlib import asynccontextmanager
import aiosqlite

db_connection = None  # will be shared across the app

@asynccontextmanager
async def app_lifespan(app: FastAPI):
    global db_connection
    db_connection = await aiosqlite.connect("example.db")  # setup
    yield {"db_status": "connected"}  # this state can be used elsewhere
    await db_connection.close()  # cleanup

app = FastAPI(lifespan=app_lifespan)

🛠 Using the Database in Your Routes

You can now use the global db_connection in your endpoints like this:

@app.get("/users")
async def get_users():
    cursor = await db_connection.execute("SELECT * FROM users")
    rows = await cursor.fetchall()
    return rows


You could also use app.state to store the connection if you want to avoid global variables:

@asynccontextmanager
async def app_lifespan(app: FastAPI):
    app.state.db = await aiosqlite.connect("example.db")
    yield
    await app.state.db.close()

Then access it in your route via request.app.state.db.

🧼 Why This Pattern Is Great

  • Keeps your setup and cleanup logic cleanly separated

  • Works perfectly with async resources like aiosqlite, httpx, or async ORMs

  • Ensures your app doesn’t start until everything is ready

🌐 Bonus: Tracking Lifespan State

You can even return a state dictionary from your yield, which is useful for debugging or passing shared config:

yield {"db_status": "connected", "env": "prod"}

And then access it in the app or routes using request.app.state if you stored it there.

🔧 More Use Cases

Besides SQLite, you can use this pattern to manage:

  • MCP server resource provisioning

  • PostgreSQL or MySQL connection pools

  • Redis or Memcached setup

  • ML model loading

  • Message broker (RabbitMQ, Kafka) clients

  • API client sessions (e.g., httpx.AsyncClient())

📌 In summary, using @asynccontextmanager with FastAPI’s lifespan lets you safely and cleanly manage async resources like an SQLite database connection. It’s a powerful pattern that keeps your startup and shutdown logic organized and robust.

Full sample code can run, contextmgmt.py:

from fastapi import FastAPI, Request
from contextlib import asynccontextmanager
import aiosqlite

@asynccontextmanager
async def app_lifespan(app: FastAPI):
    # Setup
    db = await aiosqlite.connect("example.db")
    await db.execute("""
        CREATE TABLE IF NOT EXISTS users (
            id INTEGER PRIMARY KEY AUTOINCREMENT,
            name TEXT NOT NULL
        )
    """)
    await db.commit()

    status = {"db_status": "connected", "startup_complete": True}
    app.state.db = db
    app.state.lifespan_status = status

    print("✅ Database connected and status initialized")
    yield status  # Yielding the status dictionary

    # Teardown
    await db.close()
    print("🛑 Database connection closed")

app = FastAPI(lifespan=app_lifespan)

# Add a new user
@app.post("/users")
async def create_user(request: Request, name: str):
    db = request.app.state.db
    await db.execute("INSERT INTO users (name) VALUES (?)", (name,))
    await db.commit()
    return {"message": f"User '{name}' added successfully."}

# Fetch all users
@app.get("/users")
async def get_users(request: Request):
    db = request.app.state.db
    cursor = await db.execute("SELECT id, name FROM users")
    users = await cursor.fetchall()
    return [{"id": row[0], "name": row[1]} for row in users]

# Check lifespan startup status
@app.get("/status")
async def get_status(request: Request):
    return request.app.state.lifespan_status

# Root endpoint
@app.get("/")
async def root():
    return {"message": "FastAPI with SQLite using lifespan!"}

if __name__ == "__main__":
    import uvicorn
    uvicorn.run("contextmgmt:app", host="127.0.0.1", port=8000, reload=True)
