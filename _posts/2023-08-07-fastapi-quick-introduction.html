FastAPI is a modern, fast (high-performance), web framework for building APIs with Python 3.7+ based on standard Python type hints. It is one of the fastest Python frameworks available1. The key features of FastAPI are:

  • Fast: Very high performance, on par with NodeJS and Go (thanks to Starlette and Pydantic).

  • Fast to code: Increase the speed to develop features by about 200% to 300%.

  • Fewer bugs: Reduce about 40% of human (developer) induced errors.

  • Intuitive: Great editor support. Completion everywhere. Less time debugging.

  • Easy: Designed to be easy to use and learn. Less time reading docs.

  • Short: Minimize code duplication. Multiple features from each parameter declaration. Fewer bugs.

  • Robust: Get production-ready code. With automatic interactive documentation.

  • Standards-based: Based on (and fully compatible with) the open standards for APIs: OpenAPI (previously known as Swagger) and JSON Schema.

FastAPI/Django/Flask

Django, Flask, and FastAPI are all Python web frameworks used when developing web applications. Django is a full-stack web development framework that is popular for its robust and straightforward nature. Flask is a microframework that is more flexible than Django. FastAPI is a modern, fast (high-performance), web framework for building APIs with Python 3.7+ based on standard Python type hints.

FastAPI is faster than Flask and Django in terms of performance. Flask is more flexible than Django.

Hands on Example

from fastapi import FastAPI

app = FastAPI()

@app.get("/")
async def root():
    return {"message": "Hello World"}

@app.get("/items/{item_id}")
async def read_item(item_id: int, q: str = None):
    return {"item_id": item_id, "q": q}

This code creates two routes: one for the root URL and one for URLs that include an item ID. When the user navigates to the root URL, the root() function is called and returns a JSON object with a message. When the user navigates to a URL that includes an item ID (e.g., /items/42), the read_item() function is called and returns a JSON object with the item ID and any query parameters.

q is a query parameter that can be used to pass additional information to an API endpoint. Query parameters are optional parameters that can be included in a URL after a question mark (?). They are used to filter or modify the results returned by an API endpoint.

For example, in the following URL:

/items/42?q=some-query

Start the server

FastAPI uses Uvicorn as the default server. Uvicorn is an ASGI server that can be run with a variety of different web servers.

By default, Uvicorn binds to localhost on port 8000. You can change the host and port by passing command-line arguments to the Uvicorn server.

import uvicorn
from fastapi import FastAPI

app = FastAPI()

@app.get("/")
async def root():
    return {"message": "Hello World”}

@app.get("/items/{item_id}")
async def read_item(item_id: int, q: str = None):
    return {"item_id": item_id, "q": q}

if __name__ == "__main__":
    uvicorn.run(app, host="0.0.0.0", port=8000)

What is Uvicorn

Uvicorn is a lightning-fast ASGI server that can be used to run Python web applications. It is designed to be fast and efficient, making it an ideal choice for high-performance applications.

ASGI stands for Asynchronous Server Gateway Interface. It is a standard interface between web servers and Python web applications or frameworks. ASGI is designed to support asynchronous programming, which can improve the performance of web applications by allowing them to handle more requests at the same time.

FastAPI uses ASGI as its underlying interface for handling HTTP requests and responses.
