I have a Nvidia Jetson Xavier NX from twowint https://twowinit.com/, model is T505, purchased around 2022.

It has a 21 TOPS GPU inside, we have a small team want to reuse this for GenAI edge cases.

However, as it has been longer time, the system OS version is old (ubuntu 18.04), we want to upgrade the OS into 22, via. we are successful to 20, but failed at 22, this box totally not-responsive..

Consulting with Twowint Smart AI chatbot: https://maxkb.twowinit.com/ui/chat/

It tells me two major steps:

  1. download and install SDK manager to do this task, from Nvidia website: https://developer.nvidia.com/sdk-manager

  2. put the jetson box into recovery mode (press REC, then press RES together for 2+ seconds, then release)

During step 1, I fell into the pitfall of host OS requirement.

My Jetson only can ugprade to 5.1.X version, which requires the Host OS must be Ubuntu 18 or 20.

Weird request, I only have the ubuntu 24, apparently it does not work.

Then I have two options:

1, user sdk manager docker image with CLI, tried, too difficult to use..

2, run up a virtual machine with Ubuntu.

Then I used the https://www.virtualbox.org/ to fire up the Ubuntu.

Mount the USB of T505 into the virtual machine.

Finally, follow the guide line: https://docs.nvidia.com/sdk-manager/install-with-sdkm-jetson/index.html

Remind to fire up the M2 SSD as root disk, thus have enough space to install cuda. Handy tool here: https://github.com/jetsonhacks/rootOnNVMe

and one good post here: https://blog.csdn.net/m0_53717069/article/details/128536837

Figure 1: Connected with Jetson, and starting to download and install

Figure 3: start Jetson, configure M2 disk as root (github tool), then connect via ssh from sdk manager to install CUDA.

Wow, it works now, my T505 survives and upgraded.

Finally pip install "jetson-stats" to get command "jtop" (similiar to nvidia-smi)

With 5.15 Jetpack, cuda version is 11.4. Good enough for next step.

Keep in Tune.
