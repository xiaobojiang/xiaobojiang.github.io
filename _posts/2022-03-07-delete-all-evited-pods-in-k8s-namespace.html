One day, you find there is a long list of evited pods in your kubernetes namespace. Luckily, the evited pod finally lauched on another node. what happened and how to handle these evicted pods which already stopped?

Why pods get evicted?

the pod get evicted has many potential cause.

1, node resource issue. This is the most frequent case, it can be lack of node resource, eg, Ram, CPU, ephemeral storage. The K8S cluster then will evict pods based on the priority, those pod without resource limit and require specifiction will get evited first.

2, node restart or upgrade. if user is upgrading k8s version on the node, he need to evict all pods on the node, and do the upgrade. Another case is security patch, I've encoutnered that when security patch is deployed, node will restart and pods get evicted.

What should we do to evicted pods?

Most of the time, you can ignore it, because k8s will find alternative node to launch the pod.

It still will make the "kubectl get pods" command outputs a very annoying list.

Some post claims that these pods will consume additional IP, cause the network manager running out of IP resources.

This is not correct according to my observation, if you execute 

$ kubectl get pods -n <namespace> -o wide

you will find those pods are not assigned with IP anymore, and they are actually dead. (my K8S version is 1.20)

so, except the list is annoying, there is no actual big resource occupation issue.

How to delete all evicted pods?

sometimes, the evicted pods list in the kubectl output really disturbes the monitoring of other normal pods, we want to delete these pods.

One way is that you can delete the pod one by one, with its unique id.

Is there a way to delete them all with one command? Sure, try this one:

kubectl get pod -n <namespace> | grep Evicted | awk '{print $1}' | xargs kubectl delete pod -n <namespace>

This greps out the evicted pods lines in the output, and use aws to extract the pod name, and delete these pods in last command piece.
