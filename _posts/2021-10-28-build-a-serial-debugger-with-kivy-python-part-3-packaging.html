This is the final post for writing an Kivy Serial Debugger program, you can find other two posts talking about UI design with kv lang (https://www.iotcolon.com/?p=181) and python script for interaction (https://www.iotcolon.com/?p=194)


If you finished writing your code, and want to package the code and distribute in different os platform, like android, macOS or on Windows. Kivy offers how-to on different platforms. You can find packaging for Windows, OS X, Android and IOS in the link: https://kivy.org/doc/stable/gettingstarted/packaging.html


In this post, I will show how to pack Serial Debugger code into an app and install on the macOS.


Here, assume that Kivy and its dependencies are already installed.


PyInstaller is required for this packaging purpose, you can install it via Homebrew, or get the runnable via git.
Follow the steps:

1, create a folder “packaging” and go inside

2, get the pyinstaller: 

$git clone http://github.com/pyinstaller/pyinstaller

3, create a file named “serialdebugger.spec” in this directory. You can find spec file definition here: https://pyinstaller.readthedocs.io/en/stable/spec-files.html    

Here, add following code into it: 

# -*- mode: python ; coding: utf-8 -*-


block_cipher = None


a = Analysis(['/Users/xiaobo/Git/kivy_playground/app/serial_debugger/serialdebugger.py'],
             pathex=['/Users/xiaobo/Git/packaging'],
             binaries=[],
             datas=[],
             hiddenimports=[],
             hookspath=[],
             runtime_hooks=[],
             excludes=['_tkinter', 'Tkinter', 'enchant', 'twisted'],
             win_no_prefer_redirects=False,
             win_private_assemblies=False,
             cipher=block_cipher,
             noarchive=False)
pyz = PYZ(a.pure, a.zipped_data,
             cipher=block_cipher)
exe = EXE(pyz,
          a.scripts,
          [],
          exclude_binaries=True,
          name='serialdebugger',
          debug=False,
          bootloader_ignore_signals=False,
          strip=False,
          upx=True,
          console=False )
coll = COLLECT(exe, Tree('/Users/xiaobo/Git/kivy_playground/app/serial_debugger/'),
               a.binaries,
               a.zipfiles,
               a.datas,
               strip=False,
               upx=True,
               upx_exclude=[],
               name='serialdebugger')
app = BUNDLE(coll,
             name='serialdebugger.app',
             icon='/Users/xiaobo/Git/kivy_playground/app/serial_debugger/icon.png',
             bundle_identifier=None)

In the file,：

  • A new instance of class Analysis takes the script name with full path (…/serialdebugger.py) as input. It analyze all imports and dependencies in class members named.
  • An instance of class PYZ is a .pyz archive (described under Inspecting Archives below), which contains all the Python modules from a.pure.
  • An instance of APP is built from the analyzed scripts and the PYZ archive, here an icon can also be specifiied. This object creates the executable file for mac. For windows, you shall define in EXE instance.
  • An instance of COLLECT creates the output folder from all the other parts.

 You can specify the whole path your codes are not inside the packaging directory.

4, Then run the following command:

$ pyinstaller/pyinstaller.py serialdebugger.spec

Replace the serialdebugger with your file name where appropriate, this shall give you a <yourapp>.app in the dist/ folder.

5, put the .app file into the Application folder. Then you have it, try it out.
