Our lab purchase one most up-to-date ASUS Z690 motherboard, and it has many fantastic features: support inter i9-12900KF, DDR5, and WiFi 6.

We installed latest Ubuntu 20.04 LTS on this tower server, every peripherals works fine, except WiFi6.

There is not WiFI device identified by the system.

I believed that it is due to lack of driver, I searched a lot on the internet, there is only a windows version on ASUS website, not linux one.

Finally I found a post which shares that you can upgrade your linux kernel to 5.15. it will support.

So yes, let me share the result first, upgrade the kernel really solves the problem.

What I have tried:

1, first, some post said, latest Ubuntu 21.10 has 5.15 embedded. Not true, current 21.10 comes with kernel 5.13, and it cannot find the proper WiFi driver.

2, I tried to upgrade the kernel in Ubuntu 21.10, and by solving some issues, it worked out:

Let me share some tips for kernel upgrade:

1, download the kernel packages from Ubuntu website

We will need four packages: linux header, linux headers all, linux image, linux modules.

You can find these packages with different version at: https://kernel.ubuntu.com/~kernel-ppa/mainline

I have downloaded lasted ones on 2022-1-10, version is 5.15.13 to one folder, with following commands:

wget -c https://kernel.ubuntu.com/~kernel-ppa/mainline/v5.15.13/amd64/linux-headers-5.15.13-051513-generic_5.15.13-051513.202201050731_amd64.deb
wget -c https://kernel.ubuntu.com/~kernel-ppa/mainline/v5.15.13/amd64/linux-headers-5.15.13-051513_5.15.13-051513.202201050731_all.deb
wget -c https://kernel.ubuntu.com/~kernel-ppa/mainline/v5.15.13/amd64/linux-image-unsigned-5.15.13-051513-generic_5.15.13-051513.202201050731_amd64.deb
wget -c https://kernel.ubuntu.com/~kernel-ppa/mainline/v5.15.13/amd64/linux-modules-5.15.13-051513-generic_5.15.13-051513.202201050731_amd64.deb

2, install all the packages in the folder by following command:

sudo dpkg -i *.deb

It should complete without errors, unfortunately for me, I encountered linux header generic package installation error issue.

Try 

$sudo apt upgrade

it will show the error of dependency issue libssl3.

It means that this lib has issue, I tried, it is not available by apt install.

One solution found by me is that:

2.1 download the deb file from below website:

https://packages.ubuntu.com/jammy/amd64/libssl3/download

this is the amd64 version, if we need arm version, please go upper layer and find it.

Download via: 

$ wget -c http://kr.archive.ubuntu.com/ubuntu/pool/main/o/openssl/libssl3_3.0.0-1ubuntu1_amd64.deb

2.2 then install it

via: 

$ sudo dpkg -i libssl3_3.0.0-1ubuntu1_amd64.deb

after this, install generic package again:

sudo dpkg -i linux-headers-5.15.13-051513-generic_5.15.13-051513.202201050731_amd64.deb

That is, try again with $sudo apt upgrade, you will find it works fine!

3, Then reboot your machine, it will work perfectly.

One more thing, is this package is missing, or you did not install it, when I install nvidia driver, it will not work, and $nvidia-smi command fails and showing driver version not correct.

Once we fix the generic header package installation issue, and after reboot, it will just work fine.
