let’s extend the previous blog with a real-world API integration example that uses Structured Outputs to pull data from an LLM, validate it, and store it in a database.

📖 Use Case

Imagine you’re building a support ticket triage system.

You want the LLM to:

  1. Extract key fields from incoming ticket text

  2. Classify urgency level

  3. Save results directly to your database

Without Structured Outputs, you’d risk inconsistent formatting breaking your pipeline. With .parse() and Pydantic, you guarantee valid, structured data every time.

📐 Step 1 — Define the Schema

from pydantic import BaseModel, Field

from typing import Literal

class TicketData(BaseModel):

    customer_name: str = Field(..., description="Full name of the customer")

    issue_summary: str = Field(..., description="Brief summary of the issue")

    urgency: Literal["low", "medium", "high"] = Field(..., description="Urgency level")

    contact_email: str = Field(..., description="Customer's contact email")

🤖 Step 2 — Extract Structured Data from LLM

from openai import OpenAI

client = OpenAI()

ticket_text = """

Hi, I'm Jane Doe. My server has been down for 3 hours and I can't access my client files.

Please help ASAP. Email: jane@example.com

"""
response = client.beta.chat.completions.parse(

    model="gpt-4o-mini",

    messages=[

        {"role": "system", "content": "Extract ticket details from the user message."},

        {"role": "user", "content": ticket_text}

    ],

    response_format=TicketData

)

ticket_info = response.choices[0].message.parsed

print(ticket_info.dict())

✅ The .dict() output will always look like:

{

  "customer_name": "Jane Doe",

  "issue_summary": "Server down for 3 hours, cannot access client files",

  "urgency": "high",

  "contact_email": "jane@example.com"

}

💾 Step 3 — Save to the Database

Here’s an example using SQLite for simplicity:

import sqlite3

#Connect to DB (or create if it doesn't exist)

conn = sqlite3.connect("tickets.db")

cursor = conn.cursor()

#Create table

cursor.execute("""

CREATE TABLE IF NOT EXISTS tickets (

    id INTEGER PRIMARY KEY AUTOINCREMENT,

    customer_name TEXT,

    issue_summary TEXT,

    urgency TEXT,

    contact_email TEXT

)

""")

#Insert the parsed ticket

cursor.execute("""

INSERT INTO tickets (customer_name, issue_summary, urgency, contact_email)

VALUES (?, ?, ?, ?)

""", (

    ticket_info.customer_name,

    ticket_info.issue_summary,

    ticket_info.urgency,

    ticket_info.contact_email

))

conn.commit()

conn.close()

📊 Step 4 — Querying Tickets by Urgency

conn = sqlite3.connect("tickets.db")

cursor = conn.cursor()

cursor.execute("SELECT * FROM tickets WHERE urgency = 'high'")

high_priority_tickets = cursor.fetchall()

print(high_priority_tickets)

conn.close()

Now you have a full pipeline:

LLM → Structured Outputs → Validated Pydantic model → Database storage.

🔹 Why This Works Well in Production

  • Predictable: The database always gets valid fields

  • Type-Safe: Wrong types are caught before insert

  • Maintainable: Easy to update schema as your needs change

  • Low Overhead: No regex, no fragile string parsing
