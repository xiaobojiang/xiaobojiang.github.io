To delete all docker images with the same name but different tags and keep only the newest image, you can use the following command: docker images --format='{{.Repository}}: {{.ID}} {{.Tag}} {{.CreatedAt}}' | grep "nginx:" | sort -k 2 -r | awk 'NR>1 {print $2}'| xargs docker rmi -f

Here is what each part of the command does:

  • docker images --format ' {{.Repository}}: {{.ID}} {{.Tag}} {{.CreatedAt}}': This command lists all the docker images in your system along with their repository, tag and creation date.

  • grep "TestImage:": This command filters out all the images that have the name “TestImage”.

  • sort -k 2 -r: This command sorts the filtered images by their tag in reverse order.

  • awk 'NR>1 {print $2}': This command prints out all the image IDs except for the newest one.

  • xargs docker rmi -f: This command removes all the image IDs that were printed out by the previous command.

You can also remove docker images based on name using the following command: docker rmi $(docker images 'imagename' -q)

Here is what each part of the command does:

  • docker images 'imagename' -q: This command lists all the image IDs for a given image name.

  • docker rmi $(...): This command removes all the image IDs that were printed out by the previous command.

If you want to delete all images with the same name, you can use the following command: docker image rm $(docker image ls -a | grep "imagename" | awk '{print $3}')

Here is what each part of the command does:

  • docker image ls -a: This command lists all the docker images in your system.

  • grep "imagename": This command filters out all the images that have the name “imagename”.

  • awk '{print $3}': This command prints out all the image IDs.

  • docker image rm $(...): This command removes all the image IDs that were printed out by the previous command.
