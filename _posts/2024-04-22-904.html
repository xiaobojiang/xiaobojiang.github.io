Essentially we just run the notebook on the remote in no browser mode.

$ jupyter notebook --no-browser --port=8080

Then setup up an ssh tunnel from the local machine:

$ ssh -L 8080:localhost:8080 <REMOTE_USER>@<REMOTE_HOST>

click the link with token, it will openup your default browser, and now you can see the files on the remote server.

Now you can run the book with remote resourses but with local browser to continue.

EDIT:

Running on a specific port is not necessary. The --no-browser tag is.
