After quite a bit of trial and error, I finally got an nRF52840 USB
dongle working as a BLE controller on Ubuntu using Zephyr's hci_usb
sample.

This post summarizes the working setup and the pitfalls I encountered.

Goal

Turn an nRF52840 USB dongle into a Bluetooth HCI controller:

Ubuntu / Jetson
      ↓
   BlueZ
      ↓
USB HCI
      ↓
 nRF52840
      ↓
BLE Devices

This allows Linux to use the Nordic radio for:

  • BLE scanning

  • BLE connections

  • GATT operations

  • RSSI monitoring

Environment

Hardware

  • Nordic nRF52840 Dongle (PCA10059)

Software

  • Ubuntu 22.04

  • Python 3.12

  • Zephyr SDK 0.16.5

  • Zephyr 3.7.0

Why Zephyr 3.7?

I initially started with the latest Zephyr.

Problems encountered:

  • Zephyr 4.4 requires Python ≥ 3.12

  • New Zephyr versions expect Zephyr SDK 1.x

  • Nordic tooling changed significantly

  • Flashing became unnecessarily complicated

For this use case:

Python 3.10/3.12
Zephyr 3.7
SDK 0.16.5

was much easier to work with.

Build the HCI USB Sample

Build:

west build --pristine \
  -b nrf52840dongle/nrf52840 \
  samples/bluetooth/hci_usb

The sample configuration is:

CONFIG_BT=y
CONFIG_BT_HCI_RAW=y

CONFIG_USB_DEVICE_STACK=y
CONFIG_USB_DEVICE_PID=0x000B
CONFIG_USB_DEVICE_BLUETOOTH=y
CONFIG_USB_DEVICE_BLUETOOTH_VS_H4=n
CONFIG_USB_DEVICE_INITIALIZE_AT_BOOT=n

CONFIG_SERIAL=n
CONFIG_CONSOLE=n
CONFIG_UART_CONSOLE=n

CONFIG_BT_BUF_CMD_TX_COUNT=10

Verify Build

strings build/zephyr/zephyr.elf | grep -i Bluetooth

Expected:

Bluetooth over USB sample

Flashing

I ultimately used Windows + Nordic nRF Connect Programmer because Linux
DFU tooling was inconsistent.

Flash:

build/zephyr/zephyr.hex

using Programmer.

Verify USB Enumeration

lsusb

Expected:

Bus xxx Device xxx: ID 2fe3:000b NordicSemiconductor USB-DEV

This is normal for the stock Zephyr sample.

Verify Linux Sees the Controller

bluetoothctl list

Expected:

Controller C8:67:74:73:6C:25 xiaobo-HP-Z820-Workstation [default]

A useful test:

  1. Unplug the dongle

  2. Run bluetoothctl list

  3. Confirm the controller disappears

  4. Plug it back in

  5. Confirm the controller reappears

This confirms BlueZ is using the Nordic device.

Start BLE Scan

bluetoothctl

Inside:

scan on

Expected:

Discovery started

[NEW] Device DE:99:C1:86:2B:4C Govee_H6099_2B4C
[NEW] Device 4F:53:C7:5F:22:07 HJBLE
[CHG] Device E0:E8:68:84:50:78 RSSI: -49

Better Scan Settings

Inside bluetoothctl:

menu scan
transport le
duplicate-data on
back
scan on

This gives:

  • continuous RSSI updates

  • repeated advertisements

  • behavior closer to Android BLE scanning

Python Test

Install:

pip install bleak

Example:

import asyncio
from bleak import BleakScanner

async def main():
    def callback(device, adv):
        print(device.address, adv.rssi, device.name or adv.local_name)

    scanner = BleakScanner(callback)

    await scanner.start()
    await asyncio.sleep(30)
    await scanner.stop()

asyncio.run(main())

Common Problems Encountered

SDK Version Mismatch

Zephyr 3.7 expects:

Zephyr SDK 0.16.x

Python Version Mismatch

  • Zephyr 4.4 → Python ≥ 3.12

  • Zephyr 3.7 → Python ≥ 3.8

Missing Python Modules

Example:

ModuleNotFoundError: No module named 'elftools'

Fix:

pip install -r zephyr/scripts/requirements.txt

Flashing Tool Confusion

There are multiple incompatible generations of nrfutil and
board-specific flashing methods.

Using Windows Programmer avoided all of these issues.

Final Result

The nRF52840 is now functioning as a BLE HCI USB adapter:

  • BLE scanning works

  • RSSI updates work

  • BlueZ integration works

  • Python BLE applications work

This setup is a solid foundation for:

  • Jetson AGX Orin BLE gateways

  • BLE asset tracking

  • Indoor positioning

  • Multi-device BLE monitoring

  • Robotics applications requiring BLE connectivity

The biggest lesson: verify that bluetoothctl list shows the controller and that unplugging the dongle makes it disappear.

Wow, finally I can get a dongle with nrf52840 dongle, and best part, I can connect this dongle with external antenna (which can bring a lot innovation).
