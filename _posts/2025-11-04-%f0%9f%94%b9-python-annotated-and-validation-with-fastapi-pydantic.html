What is Annotated?

  • Introduced in PEP 593.

  • Lets you attach metadata to type hints.

  • Python itself does not enforce these rules — it just stores them.

  • Frameworks (like FastAPI and Pydantic) read the metadata and act on it.

At the Python level:

Annotated[BaseType, *metadata]

…is just a container.

Python stores both parts (base type and metadata), and the framework decides how to interpret that metadata.

So:

#Pydantic interprets Field(...)

Annotated[str, Field(min_length=3)]

#LangChain interprets InjectedToolCallId

Annotated[str, InjectedToolCallId]

Both are syntactically identical but semantically unrelated — the framework reads the metadata differently.

Detailed examples in different frameworks:

Example 1: Python alone (no enforcement)

from typing import Annotated, get_type_hints

MyInt = Annotated[int, "must be positive"]

def foo(x: MyInt): 
    return x

print(foo(-5))              # Works fine

print(get_type_hints(foo))  # {'x': typing.Annotated[int, 'must be positive']}

👉 Python keeps the "must be positive" note, but doesn’t validate.

Example 2: FastAPI

from typing import Annotated
from fastapi import FastAPI, Query
app = FastAPI()
@app.get("/items/")
def read_items(
    q: Annotated[str | None, Query(min_length=3, max_length=50)] = None
):
    return {"q": q}

  • If you send /items/?q=hi, FastAPI returns 422 Validation Error.

  • FastAPI uses Annotated metadata (Query(...)) for validation + docs.

Example 3: Pydantic v2

from typing import Annotated

from pydantic import BaseModel, Field

class User(BaseModel):

    name: Annotated[str, Field(min_length=3, max_length=50)]

    age: Annotated[int, Field(gt=0, lt=120)]

print(User(name="Alice", age=25))   # ✅ OK

print(User(name="Al", age=-1))      # ❌ ValidationError

  • Pydantic enforces rules from Field(...).

  • But text-only metadata is ignored:

age: Annotated[int, "must be > 0"]  # ⚠️ No validation

✅ Summary

  • Python itself: Annotated = metadata only, no enforcement.

  • FastAPI: uses metadata (Query(...)) for request validation & docs.

  • Pydantic: uses metadata (Field(...), validators) for model validation.

  • Text notes inside Annotated are ignored by Pydantic — only structured objects matter.

👉 Use Annotated when you need type + validation/metadata in a single, clean definition.
