Infrasture as code, this concept enables that infrastructure resource eg, VPN, EC2, security groups, RDS, EKS can be managed and deployed with code. This can helps OP guys quickly replicate infrastructure quickly.


AWS already provides a way called cloudformation, by writing the xml, user can define and deploy with this concept. However, writing cloudformation from scratch is really painful, and this leads to hundreds of lines even for a simple feature.

Cloud developer kit (CDK) was release by AWS on 31 July 2018. CDK was written in Typescript, with JSII it can support Python and other languages.


With CDK, user can write few lines to bring up a VPC, EC2, and a lot of things.


I started CDK usage from March 2021, together with ansible, it helped me in the project to quickly deploy environment and applications.
This is the first post on installation and project creation with CDK python.


Environment preparation
Before we start exploring the CDK features, you need to install some necessary packages:

  • Python 3
  • pip
  • virtualenv
  • Nodejs
  • aws-cdk

I am using MacBook, you may need to adpat a little for your OS.


Let’s start from Python, it is pre-installed with python2.7 in Mac OS. Python3 is more recommended since Python2 is a sunsetting product, and it does not receive any bug fix and changes.


However, if you use Homebrew to install python3 and let homebrew to maintain python3 updates, you will face night-mares of package management and dependencies.


I use PyEnv (https://github.com/pyenv/pyenv) to manage python versions, I may write one separate post on how-to.


Let’s assume you already have your python3 and pip ready on your computer.


Next is to install virtualenv for python, this can help you manage different package and version for different projects. It is simple to install: 

$ pip install virtualenv


All AWS CDK applications require Node.js 10.13 or later, even if you work in Python, Java, or C#. Here, let’s install the latest version of Node.js Download and install the corresponding package from: https://nodejs.org/en/download/current/


Use following commands to verify nodejs installation, you should get something similar:

After installing Node.js, install the AWS CDK toolkit (you can use cdk command then): $ sudo npm install -g aws-cdk Verify your installation by:

Now everything is ready, you can start your first project. $mkdir my-project $cd my-project $cdk init app —language python Now you can see such a folder:

After initializing the project, activate the project's virtual environment. This allows the project's dependencies to be installed locally in the project folder, instead of globally. source .venv/bin/activate
Now, you can install CDK libraries for use.

For example, you can install aws-ec2. 

$ pip install aws-cdk.aws-ec2

This will also download linked packages:

Suggest manually update the requirement.txt inside the folder, avoid use pip freeze or piperqs, this will bring you two many circular depencies.

Add lines like:
aws-cdk.aws-ec2==1.118.0
Later, you can user pip install -r requirements.txt to install packages.


Now, edit the my_projects/my_project_stack.py, let's create a VPC: 

from aws_cdk import core as cdk
from aws_cdk import aws_ec2 as ec2
class VpcStack(cdk.Stack):

    def __init__(self, scope: cdk.Construct, construct_id: str, cidr_range, **kwargs) -> None:
        super().__init__(scope, construct_id, **kwargs)


        self.vpc = ec2.Vpc(self,
            'brightsite-vpc', # VPC id
            cidr=cidr_range, # CIDR
            max_azs=3,  # 3 aza
            nat_gateways=1 # create NAT Gateway
        )

Here, I strongly recommend Visual Studio Code + AWS Toolkit.

It can help you with hints, and autocomplete, it really improves efficiency!


Before you run the upper code, you need to have an AWS profile provisioned. $aws configure —profile test_profile # fill the account id and key, zones, and format info. 

$ export AWS_PROFILE=test_profile


Then run the code to create a VPC right now, go the my_project root folder: 

$ cdk bootstrap 
$ cdk deploy


You can optionally use “cdk diff” to see the difference.

And the generate Cloudformation code is located in the cdk.out folder.
