Before talking the topic as the headline, first I would like to share available ADC sampling triggers in STM32 platform:

- software polling

- DMA

- Timer trigger ADC + DMA transfer

In principle, STM32 ADC has following working mode: single sampling or continuous sampling.

It can be combined for use,  for example, software polling + single sampling.

So, my use case it that I need STM32F1 board to sampling at 50K frequency, and MCU focuses on data processing for every 4ms. And data processing takes a lot of time, and during this time, sampling shall not be interrupted.

Let’s first look at triggering methods, software polling needs continues trigger in the firmware, which is not good for my case.

DMA is good for continuous transferring from peripheral to memory in circular mode, however, the sampling frequency is hard to control.

Here, the idea is make best use of DMA + ADC to do continuous sampling, it requires barely any CPU time. And use a timer, eg TIM3 to provider trigger, this can ensure accurate sampling frequency.

I will start with an example of 100Hz, 2 channel sampling. 

The project configuration is done with STM32CubeMX tool.

Let’s first configure the ADC.

In below pic, we enable IN6 & IN7 as input channel.

Sample mode is Scan Conversion.

Setting of conversionMode is: 

Enable regular conversion

Number of conversion is 2, since we have two inputs 

External trigger source is: Timer 3 Trigger out event (there are other options, this one is quite convenient to use)

In Below two Ranks: rank 1, select IN6 as input, and configure sampling clocks cycles (7.5), rank2, select IN6 as input, and configure sampling clocks cycles (7.5). You can configure sampling clocks cycles based on your needs

In the “DMA Settings” tab, enable DMA

Add ADC1 direction is “Peripheral to Memory”.

In the DMA setting, mode must be “Circular”, otherwise it will stop once the buffer is full.

My STM32F1 ADC precision is 12bits, so Data Width we choose Half Word

CubeMX will also auto enable DMA interrupt in NVIC Setting, that is great! 

Finally, setting up the TIM3 as trigger source

In the board, APB1 Timer clocks frequency is 24MHz.

The ADC clock frequency must not be larger than 12MHz.

Here, in the example, the sampling frequecy is 100Hz

Thus 24M / Prescaler (2400) / count limit (100)  = 100

Set these values, and enable auto reload

Last thing, set the Trigger Event to “Update Event”, everytime TIM3 reloads, it will trigger TIM3 TRGO event for ADC use.

Now let CubeMX generate code for you.

In the opened project file, edit main.c

Create the buffer and flag indicating DMA complete

Buffer size shall be adjusted for real case.

In the buffer, first element is RANK 1, second is RANK 2, if you buffer is larger, third and fourth is again RANK1, RANK2.

/* USER CODE BEGIN PV */

#define ADC_BUFLEN 2
uint16_t adcBuf[ADC_BUFLEN] ={0};   //For ADC samples.

uint8_t ADC_FLAG;
#define TRUE 1
#define FALSE 0
/* USER CODE END PV */

In the DMA interrupt call back, set the ADC flag.

Here, I use the DMA full callback, you can also use half full callback.

void HAL_ADC_ConvCpltCallback(ADC_HandleTypeDef* hadc1){

  ADC_FLAG=TRUE;
}

Add some testing code, in the while loop.

Print out the number of sampling in one second, and some latest values:

while (1)
  {
    /* USER CODE END WHILE */

    /* USER CODE BEGIN 3 */
    if(ADC_FLAG)
    {
      ADC_FLAG=FALSE;
      ++i;
      if(HAL_GetTick() - current_ticks >= 1000)
      {
        current_ticks = HAL_GetTick();
        sprintf(msgbuf, "%d: ADC6=%d, ADC7=%d\r\n", i,
                (int)adcBuf[0], (int)adcBuf[1]);
        HAL_UART_Transmit(&huart1, msgbuf, strlen(msgbuf),10);
        i=0;
      }
    }
  }

Now, Build the project, and monitor the data in the serial debugger:

Perfect: every second, sampled 100 times, and values are right to what you set on the pin.
