Sometimes a command that worked for years suddenly starts failing after a laptop upgrade.

That happened to me recently while copying files to an embedded Linux device. The target was reachable, SSH login worked, and the path was correct:

ssh root@192.168.1.10

No problem.

But when I tried to copy a file with scp, the transfer failed. The confusing part was that nothing had changed on the device side. The device still had the same SSH server, the same /data/ directory, and the same root login.

The real change was on my host machine.

Starting with OpenSSH 9.0, scp uses SFTP by default instead of the older SCP/RCP-style protocol. That is usually a good thing because SFTP has safer and more predictable filename handling. But on some older devices, minimal Linux systems, recovery images, vendor firmware, or embedded targets, SFTP may not be available or may not behave the way modern scp expects.

The fix is simple: force legacy SCP mode with -O.

scp -O localfile root@192.168.1.10:/data/

Copy from Local Machine to Target

Use this when you want to send a file from your computer to the device:

scp -O localfile root@192.168.1.10:/data/

Example:

scp -O zeroclaw root@C42996CA26ED:/data/

This copies the local file zeroclaw into /data/ on the target device.

Copy from Target Back to Local Machine

Use this when you want to pull a file from the device:

scp -O root@192.168.1.10:/data/file.txt .

The final . means “copy it into the current local directory.”

Copy a Directory Recursively

For folders, add -r:

scp -O -r mydir root@C42996CA26ED:/data/

This copies the entire local directory mydir to /data/ on the target.

Use a Specific SSH Port

Remember that scp uses uppercase -P for the port:

scp -O -P 2222 file.txt root@192.168.1.10:/data/

That is different from ssh, where the port option is lowercase -p.

Why -O Works

The -O option tells OpenSSH scp to use the old SCP protocol instead of the newer default SFTP mode. This is useful when the remote target only supports the legacy behavior, which is common on older firmware, stripped-down embedded systems, or devices with limited SSH/SFTP support.

In other words:

scp file root@target:/path/

may now mean “copy using SFTP-style behavior,” while:

scp -O file root@target:/path/

means “copy using the old SCP/RCP-style protocol.”

Quick Cheat Sheet

# Copy one file to target
scp -O file.txt root@192.168.1.10:/data/

# Copy one file from target
scp -O root@192.168.1.10:/data/file.txt .

# Copy a directory to target
scp -O -r mydir root@192.168.1.10:/data/

# Copy using a custom SSH port
scp -O -P 2222 file.txt root@192.168.1.10:/data/

Takeaway

When ssh works but scp suddenly fails after an OpenSSH upgrade, do not immediately blame the network, the password, or the target path.

Try this first:

scp -O ...

For modern servers, SFTP mode is the better default. For older embedded devices, -O can be the small compatibility flag that saves a lot of debugging time.
