To manage AWS CLI configurations and profiles, follow these steps:

🔍 List Existing AWS CLI Profiles

To view all configured profiles:

aws configure list-profiles

This command lists all profiles defined in your AWS CLI configuration files.

🛠️ Create a New Named Profile

To create a new profile with a specific name:

aws configure --profile my-profile-name

You’ll be prompted to enter:

  • AWS Access Key ID

  • AWS Secret Access Key

  • Default region name (e.g., us-east-1)

  • Default output format (json, text, or table)

This creates or updates entries in the /.aws/credentials and /.aws/config files.

🔄 Use a Specific Profile in AWS CLI Commands

To use a specific profile for a single command:

aws s3 ls --profile my-profile-name

To set a profile as the default for your session:

export AWS_PROFILE=my-profile-name

This environment variable sets the default profile for all subsequent AWS CLI commands in the current session.

📁 AWS CLI Configuration Files

AWS CLI stores configuration in two main files:

  • ~/.aws/credentials: Contains access keys for profiles.

  • ~/.aws/config: Contains region and output format settings for profiles.

Each profile has its own section in these files.

By using named profiles, you can efficiently manage multiple AWS accounts or roles within the AWS CLI. If you need further assistance with AWS CLI configurations or profiles.
