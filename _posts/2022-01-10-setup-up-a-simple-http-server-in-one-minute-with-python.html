Sometimes, when I setup a new network, eg a VPN tunnel, or a server is deployed in some VPC, I want to know that is the network or firewall properly configured. 

The easiest way before deploying the service, is to setup a simple http service. Then with tools like telnet or curl, the service reachability can be easily verified.
I would like to provide some methods based on Python, which can help you quickly setup a server with few single lines.


Method 1: SimpleHTTPServer

What is it: This is the built-in web server lib in Python: SimpleHTTPServer, which comes with Python is a simple HTTP server that provides standard GET and HEAD request handlers.
How to use it:

$ python -m SimpleHTTPServer

This starts a HTTP server on port 8000 (which is the default port)

Be cautious: This will now show the files and directories which are in the current working directory.

You can also change the port to other (eg.8080), with following command:

$ python -m SimpleHTTPServer 8080

If you face the error: “No module named SimpleHTTPServer”
This is because SimpleHTTPServer is merged with http.server in python3. Please use the below command, replace SimpleHTTPServer with http.server:

$ python3 -m http.server 8080

Test it:Just use your favorite browser, type: https://<server ip>:<port>You shall see the directories.

Or with curl:

Comment:

It is easy, but only for testing purpose, and it will expose your files for others, be cautious.

Method 2: flask
What is it?Flask is a web framework, it’s a Python module that lets you develop web applications easily.Flask is also easy to get started with as a beginner because there is little boilerplate code for getting a simple app up and running.
How to use it?

Installation:

$ pip install flask

We will build a simple “Hello World” web application.

Prepare a file with name “app.py”
In the file, place few lines of code:

from flask import Flask

app = Flask(__name__)

@app.route("/")
def hello_world():
    return "<p>Hello, World!</p>"

Run the server:

With command 

$ flask run —-port 8080
* Environment: production
WARNING: This is a development server. Do not use it in a production deployment.
Use a production WSGI server instead.
* Debug mode: off
* Running on http://127.0.0.1:8080/ (Press CTRL+C to quit)
127.0.0.1 - - [05/Jan/2022 11:03:33] "GET / HTTP/1.1" 200 -


Test it:Try the http://localhost:8080

 in the browser, you shall get the “Hello World!"

And this also works for curl:

Comments:

Using flask requires few more lines of code, but it can be easily extended to more rich features, and support complicated http request.
These are the two methods to setup a simple http server in one minute.
