Since its very early launch of openclaw (with earlier name, openclaud, moltbot), I start to setup a RPI5 as host to make it live with (lol). There are so many well documented articles about how to setup this "claw" in a host, I will not duplicate here. But, if you have any question on using this on RPI5, just let me your issue or puzzle, I can share more.

This post is really about a niche setup about using openclaw. In order to make it safe running, I set it up only in local network for access. I use a jump host staff to visit it for cli. However, then the IP address is not fixed, I get tired of hunting and remembering IP, I prefer to use DNS name (eg, openclaw.local) for visit it in local network.

mDNS comes to help.

On Linux, the most common mDNS implementation is Avahi (Bonjour-compatible). It publishes your host/service info on the LAN and lets other devices resolve *.local names without a central DNS server.

1) Set the hostname to openclaw

sudo hostnamectl set-hostname openclaw

Update /etc/hosts so it matches (swap out the old hostname if needed):

sudo nano /etc/hosts

Make sure you have something like:

127.0.0.1   openclaw

2) Install and enable Avahi (mDNS)

sudo apt update
sudo apt install avahi-daemon libnss-mdns
sudo systemctl enable --now avahi-daemon

libnss-mdns hooks mDNS into Linux name resolution so .local lookups actually work.

3) Ensure .local name resolution is enabled (NSS)

Check the hosts: line:

grep '^hosts:' /etc/nsswitch.conf

A commonly recommended Debian-family config looks like:

hosts: files mdns4_minimal [NOTFOUND=return] dns mdns4

This ordering makes .local resolve via mDNS before falling back to normal DNS.

If your hosts: line doesn’t include mdns / mdns4_minimal, edit it:

sudo nano /etc/nsswitch.conf

4) Restart and test

sudo systemctl restart avahi-daemon

From another device on the same LAN/Wi‑Fi:

ping openclaw.local
# or
ssh pi@openclaw.local

Optional (handy debugging tools):

sudo apt install avahi-utils
avahi-resolve -n openclaw.local
avahi-browse -a -t

Avahi utilities can help you confirm what’s being advertised/discovered. citeturn0search1turn0search0

Notes / gotchas

  • Same subnet only: mDNS is generally link-local; it usually won’t traverse VLANs/subnets unless you run a reflector.

  • Port & multicast: mDNS uses UDP 5353 and multicast (IPv4 224.0.0.251, IPv6 ff02::fb).

  • Name conflicts: if another device already has the same name, you may see openclaw-2.local.
