Continued with https://www.iotcolon.com/?p=1126, actually several common Python frameworks and libraries, especially in the web, async, and dependency injection ecosystems, support a concept of lifespan, which typically refers to startup and shutdown hooks for managing resources.

Here’s a breakdown of key libraries that support lifespan-style parameters or concepts:

✅ 1. FastAPI

✔️ Supports lifespan parameter (since FastAPI 0.95)

from fastapi import FastAPI

from contextlib import asynccontextmanager

@asynccontextmanager

async def lifespan(app: FastAPI):

    # Startup logic

    print("Startup: connect to DB")

    yield

    # Shutdown logic

    print("Shutdown: disconnect DB")

app = FastAPI(lifespan=lifespan)

Used to manage DB connections, background tasks, cache, etc.

✅ 2. Starlette

FastAPI is built on Starlette, which also supports lifespan hooks:

from starlette.applications import Starlette

from contextlib import asynccontextmanager

@asynccontextmanager

async def lifespan(app):

    print("Start services")

    yield

    print("Stop services")

app = Starlette(lifespan=lifespan)

✅ 3. AnyIO/ Trio

Both offer structured concurrency and resource scoping. While they don’t use the word “lifespan”, the pattern is the same.

import anyio

async def main():

    async with anyio.create_task_group() as tg:

        tg.start_soon(worker)

async def worker():

    print("Running background task")

This is lifespan-like because the TaskGroup or nursery defines the lifetime of tasks.

✅ 4. Typer (CLI framework by FastAPI author)

It uses FastAPI-style hooks if you embed a FastAPI app or use async commands. It doesn’t have a built-in lifespan, but you can simulate startup/shutdown logic in a CLI context.

✅ 5. Dependency Injection Libraries

Like punq, wired, or lagom, often usecontext-managed containersor lifecycle scopes (singleton, transient, etc.). For example:

with container.scoped() as scope:

    service = scope.get(MyService)

This is a form of lifespan control — for DI containers.

✅ 6. AsyncExitStack  / @asynccontextmanager

Used directly or indirectly in many libraries to manage resource lifetimes (e.g., HTTP clients, DB connections, locks).

✅ 7. Uvicorn / ASGI Servers

They honor lifespan protocol for startup/shutdown of ASGI apps (like FastAPI/Starlette). You can disable it with --lifespan off, or rely on it for graceful resource handling.

🚫 What Doesn’t Use 

lifespan

  • Traditional WSGI apps (e.g., Flask, Django) — unless manually adapted

  • Sync scripts or tools (unless you manually use try/finally or contextlib)
