In previous post, I shared the docker secret to protect sensitive information.

We learned its application and limitations.

In this post, let's try podman secret.

I verified feature of secret on podman version: 4.1.1. if your version is newer, please let me know anything improved in the reply.

Podman secret create

$podman secret create my_secret tc_datadir/certs/client.crt
$podman inspect secret my_secret

Let's inspect the path, and content inside.

Apparantly, it is base64 encoded file. Let's decode and check the content.

Oh, that's not nice, it is just encoded not encrpyted.

Mount secret and use

https://docs.podman.io/en/latest/markdown/options/secret.html

Secret are mounted at default location /run/secrets/ inside the container.

we can specify the mounted target name by target=<target file name>

$ podman run -it -d --secret my_secret,target=client.crt alpine

Now, you can find it in /run/secrets/client.crt

If we want to mount the secret at specific location. It also possible, specify the target with full path

$ podman run -it -d --secret my_secret,target=/env/certs/client.crt alpine

Well, that's it.

Compared to docker secret, podman secrets:

1, are natively supported, in podman run and compose mode.

2, are not encrupted. Use can get more details with podman secret inspect command.
