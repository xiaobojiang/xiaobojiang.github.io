The “temperature” parameter in a large language model (LLM) controls how the logits (raw scores) are converted into probabilities and thus influences how randomly the model picks its next token. Here’s a breakdown:

🧮 How temperature works

  • A model produces logits z_i for each possible next‐token i.

  • The usual softmax turns logits into probabilities: $$P(i) = \frac{\exp(z_i)}{\sum_j \exp(z_j)}$$

  • With temperature T, you scale the logits before softmax: $$P(i) = \frac{\exp(z_i / T)}{\sum_j \exp(z_j / T)} $$

  • Interpretation of T:

    • If T = 1: it’s the standard softmax. 

    • If T > 1: the distribution becomes flatter. Less difference between high and low logits → more randomness → more variety. 

    • If T < 1: the distribution becomes sharper. The highest logit dominates more → the model is more “confident” / deterministic. 

    • In the limit as T \to 0: the distribution tends toward a one‐hot: basically always pick the token with the highest logit. 

✅ So, when you set temperature = 0…

In theory: yes — when T \to 0, the model’s probability distribution collapses to the arg max (i.e., the token with the highest logit gets probability → 1, others → 0). For example:

“T → 0: Distribution becomes one‐hot (argmax)” 

“T = 0: Deterministic, always selects the highest probability token” 

In practice: however, many LLM APIs do not allow exactly 0, or treat extremely small values (~0.01) as “greedy” sampling mode. Using “temperature = 0” often means “greedy decoding” (always pick the highest‐probability token) rather than “strictly mathematically dividing by zero”.

⚠️ Things to watch

  • Even if you set a very low temperature, whether you always pick the highest‐probability token depends also on decoding strategy (greedy vs sampling) and whether there are additional constraints (like top-k, top-p filters, repetition penalties). For example, even at low temperature you might still have top-k sampling which might add randomness. 

  • Setting temperature too low can make the output boring, repetitive, and stuck in safe responses. Setting it too high can make it nonsensical. It’s a trade‐off between coherence and creativity. 

📋 Summary

  • Temperature controls randomness in token selection by scaling logits before softmax.

  • Lower T → more deterministic (sharper distribution). Higher T → more random, exploratory (flatter distribution).

  • Setting T=0 (or very close) → effectively greedy decoding → pick the single token with highest score (in ideal math).

  • But in real‐world LLM systems: Check how the “temperature=0” mode is implemented (often means greedy decoding + no randomness).

  • Use temperature as a knob: if you want very reliable, factual output → use low T; if you want creative generation → use higher T.
