Target dilema:

I want to stop pods of one deployment in K8S, and execute some actions on other resources (eg, RDS), and then resume the pods again.

Solution not so good:

Totally delete the deployment, do the actions and start the deployment again.

#delete the deployment
$ kubectl delete deployment <deployment-name> -n <namespace>
#start again
$ kubectl apply -f deployment-name.yaml

Downside: to start the deployment, we need to prepare the yml file, if you are using templating tools to generate yml, it will require a lot of commands.

Here, I found a better one and would like to share with you

Step 1: reduce the replicas of this deployment to 0

$ kubectl scale deployment/<deployment-name> -n <namespace> --replicas=0

Step 2: do the actions

Step 3: scale up the deployment replicas to desired count

$ kubectl scale deployment/<deployment-name> -n <namespace> --replicas=n
