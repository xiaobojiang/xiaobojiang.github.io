Here, nginx mounted a folder of certs inside.

And externally, we use a certbot container to create certs.

And then cp files, of course, they can share the same mounting folder if applicable.

Ansible script

- name: provision letencrypt working data folder
file:
path: '{{ ansible_env.HOME }}/letsencryptdir'
state: directory
mode: '0755'

- name: provision app cfg folder if not yet
file:
path: '{{ ansible_env.HOME }}/s_cfg/cert'
state: directory
mode: '0755'

- name: install certbot via pip
pip:
name: certbot
state: present

- name: install crontab and crond via cronie on aws ec2
package:
name: cronie
state: present
become: yes

- name: start crond and enable
ansible.builtin.systemd:
state: started
enabled: yes
name: crond
become: yes

- name: provision shell file to server
template:
src: ./templates/renewcert.sh
dest: '{{ ansible_env.HOME }}/renewcert.sh'
mode: 'u+x'

- name: create crontab to run on hour 3 on every sunday
cron:
name: "renew cert"
minute: "0"
hour: "3"
weekday: "0"
job: "{{ ansible_env.HOME }}/renewcert.sh >> {{ ansible_env.HOME }}/cronlog.log 2>&1"

Intall certbot

$ pip install certbot

Prepare a script.

#!/bin/sh
cert_dir="{{ ansible_env.HOME }}/letsencryptdir"
dest_dir="{{ ansible_env.HOME }}/s_cfg/{{ cert_dir }}"

if [ $( docker ps -a | grep nginx | wc -l ) -gt 0 ]; then
docker stop nginx;
fi


if [ -z "$(ls -A $cert_dir/live 2>/dev/null)" ]; then
echo "Certificate directory is empty or not present. Generating new certificate..."
docker run --rm -p 80:80 -v $cert_dir:/etc/letsencrypt certbot/certbot certonly --standalone --rsa-key-size 4096 -d {{ app_host }} --cert-name certs --register-unsafely-without-email --agree-tos
else
echo "Certificate renewal found. Running Certbot renew..."
docker run --rm -p 80:80 -v $cert_dir:/etc/letsencrypt certbot/certbot renew --non-interactive --quiet
fi

sudo cp $cert_dir/live/certs/cert.pem $dest_dir/{{ site_cert_name }}
sudo cp $cert_dir/live/certs/privkey.pem $dest_dir/{{ site_key_name }}

docker start nginx

And it will run with cron job, on 3am everyday.
