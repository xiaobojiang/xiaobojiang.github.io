Adding another tech note on using ZED depth camera on Jetson Orin AGX.

Overview

This guide explains how to install the ZED SDK on Jetson AGX Orin using
the existing CUDA from JetPack.

Key Principle

JetPack (L4T) → CUDA → ZED SDK

  • Do NOT install CUDA manually

  • Match ZED SDK with JetPack (L4T)

Steps

1. Check system info

cat /etc/nv_tegra_release

# R36 (release), REVISION: 4.7, GCID: 42132812, BOARD: generic, EABI: aarch64, DATE: Thu Sep 18 22:54:44 UTC 2025
# KERNEL_VARIANT: oot
TARGET_USERSPACE_LIB_DIR=nvidia
TARGET_USERSPACE_LIB_DIR_PATH=usr/lib/aarch64-linux-gnu/nvidia

---
How to interpret:
    •   R35.x → JetPack 5.x
    •   R36.x → JetPack 6.x
Final Version:
    •   L4T: 36.4.7

nvcc --version
lsb_release -a
uname -m

2. Verify CUDA

nvidia-smi
dpkg -l | grep cuda

3. Install dependencies if needed

sudo apt update sudo apt install nvidia-jetpack-dev #nvidia-jetpack

4. Download ZED SDK

5. Install

5.1 install download file

cd ~/Downloads chmod +x ZED_SDK_Tegra_L4TXX.X_vX.X.X.run
./ZED_SDK_Tegra_L4TXX.X_vX.X.X.run

5.2 run diagnostic

Then run ZED_Diagnotic
if you notice that "GMSL driver is missing or not installed"
do not worry, check your zed camera model:
If you are using:
• ZED 2 / ZED 2i / ZED Mini (USB)
👉 ✅ You can IGNORE this warning

If you are using:
• ZED X / ZED X Mini (GMSL)
👉 ❌ Then this is a real issue

5.3 optimize models

in the same diagnostic window, click AI model tab.
And select necessary models (eg, depth related) and optimize
it will download models and do necessary optimization.

6. Test

ZED_Explorer zed_depth_viewer

Notes

  • Always match L4T version

  • Use built-in CUDA from JetPack
