Recently, I've setup one Ubuntu to run Jenkins in the cooperate network.

By default, once the Ubuntu moved to intranet, it cannot reach out to public internet anymore. However, I need this Ubuntu to pull git code, install packages and pull libraries externally, network is a mandatory.

I asked IT for this, and get a proxy. Long journey begins to configure Ubuntu, and finally it runs as expected.

Here, I would like to share a few points, maybe it can also help if you also face this situation.

Add proxy for http and https

Add few lines in ~/.bashrc or ~/.zshrc, it depends on your shell option

format is 

export http_proxy=<host>:<port>

We can also setting urls that bypasss proxies, for example, some intranet services can only be visited without proxy, add them into the no_proxy list, and seperate by comma.

Then execute below command to make it effective.

$source ~/.zshrc

this shall make your curl, wget command works fine now.

Add proxy for your web Browser

However, upper setting won't work for web browser.

few additional setting required:

1, in FireFox setting page, search Proxy, and in the configuration tab, select "Use System Proxy Setting"

2, in Ubuntu's setting's "Network Proxy", check "Manual" mode, and fill the proxy address, and no proxy address, definition is same as http proxy in shell.

Now, we shall be able to browse website in Firefox now.

Add proxy for apt

apt is the tool for package management, in order to download or update package online, proxy configuration is need in intranet.

1, Create a new file at /etc/apt/apt.conf.d/proxy.conf

2, Add this line for anonymous proxy: 

Acquire::http::Proxy "http://192.168.1.12:9480”; 

For proxy with username and password:

http://[username]:[password]@[proxy-webaddress]:[port]

Add proxy for Docker

Proxy for docker daemon

If we do not configure proxy for Docker Daemon, then it is not possible to use search, pull, push command.

Configuration steps:

# STEP1-create a folder

[root@localhost home]# sudo mkdir -p /etc/systemd/system/docker.service.d

# STEP2-create proxy file for http and https

[root@localhost home]# cat /etc/systemd/system/docker.service.d/http-proxy.conf

[Service]

Environment="HTTP_PROXY=http://192.168.1.12:9480”

Environment="HTTPS_PROXY=http://192.168.1.12:9480"

#this is the setting for websites that we do not want proxy

Environment="NO_PROXY=localhost,127.0.0.1,docker-registry.example.com,.corp"

[root@localhost home]# cat /etc/systemd/system/docker.service.d/https-proxy.conf

[Service]

Environment="HTTPS_PROXY=http://192.168.1.12:9480"

# STEP3-reload to make changes effective

[root@localhost home]# sudo systemctl daemon-reload

# STEP4-retart Docker

[root@localhost home]# sudo systemctl restart docker

Cool, now we shall be able to pull and push images.

Proxy for docker containers

If your container (not daemon) needs to use an HTTP, HTTPS, or FTP proxy server, you can configure it

On the Docker client, create or edit the file ~/.docker/config.json in the home directory of the user that starts containers.

{
 "proxies":
 {
   "default":
   {
     "httpProxy": "http://192.168.1.12:9480",
     "httpsProxy": "http://192.168.1.12:9480",
     "noProxy": "*.test.example.com,.example2.com,127.0.0.0/8"
   }
 }
}

OK, that's it, I configure those settings, and now my machine works as expected.
