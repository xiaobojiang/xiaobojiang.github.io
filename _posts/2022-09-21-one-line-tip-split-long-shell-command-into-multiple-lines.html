One line tip, one minute Fix!

This tip is about split long shell command into lines, thus to enhance readability.

One line tip:

split the command by add backslash (\) and enter new line without space on each line end

Example:

$ cat test.sh                                                                                                                      [14:02:55]
echo "The File Name is: $0"
echo "The First argument is: $1"
echo "The Second argument is: $2"

Original line:

mvn sonar:sonar -Dsonar.projectKey=${sonarname} -Dsonar.host.url=${SONARQUBE_URL} -Dsonar.login=${sonarkey}

splited with tip

$ sh test.sh "1"\
 "2"
The File Name is: test.sh
The First argument is: 1
The Second argument is: 2

Note: do not insert any space after backslash, it will be taken as new command

$ sh test.sh "1"\ #there is a space
 "2"
The File Name is: test.sh
The First argument is: 1 
The Second argument is: 
zsh: command not found: 2

Why it will work:

If a \<newline> pair appears, and the backslash is not itself quoted, the \<newline> is treated as a line continuation (that is, it is removed from the input stream and effectively ignored).
