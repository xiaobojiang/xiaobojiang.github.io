Please find consolidated guide on setting up Client Credentials flow in Keycloak, configuring token expiration, defining and validating aud, and securing the flow end-to-end:

🏷️ 1. Create a Realm & Client Credentials‑Enabled Client

  1. Create a realm(e.g., MCP) in the Keycloak Admin Console – or usekcadm.sh:

kcadm.sh config credentials --server http://<host>:8080/auth --realm master --user admin --password <pwd>

kcadm.sh create realms -s realm=MCP -s enabled=true

  1. Add a client in the MCP realm:

    • Go to Clients → Create

    • Set Client ID(e.g., mcp-service)

    • ChooseProtocol: openid-connect

    • ClickSave

    • In Settings, enable:

      • Access Type: Confidential

      • Client Authentication: On

      • Service Accounts Enabled: On

    • Turn off:

      • Standard Flow

      • Implicit Flow

      • Direct Access Grants

      • OAuth2 Device Authorization

      • OIDC CIBA GrantsThese ensure only client–credentials is active.

  2. Copy the Client Secret in the Credentials tab.

⏰ 2. Set Token Expiration

  1. Realm-wide default:

    • Navigate to Realm Settings → Tokens

    • Adjust Access Token Lifespan(e.g. 5 minutes)This is the default for all clients.

  2. Per-client override:

    • In your client’s Advanced tab, set Access Token Lifespan(e.g. 2 h)Short lifetimes reduce token misuse exposure.

🎯 3. Control  aud with Custom Audience

By default, Keycloak may auto-include account in aud, or omit aud for service-account tokens. To manage this:

  1. Disable full scope inheritance(avoids auto-including account):

    • Go to Clients → your client → Client Scopes → <client-id>-dedicated

    • Under Scope, turn offFull Scope Allowed This stops default audiences from being added  .

    •  

  2. Add a custom Audience Mapper:

    • Within -dedicated scope, go to Mappers → Create

      • Mapper Type: Audience

      • Included Custom Audience: e.g. MCP

      • EnableAdd to access token

    • Save it – now aud: "MCP" appears in service-account tokens (access tokens)  .

🔐 4. Request a Token (Client Side)

Using Python + Authlib:

from authlib.integrations.requests_client import OAuth2Session

client = OAuth2Session(client_id="mcp-service", client_secret="SECRET", scope="MCP")

token = client.fetch_token(url="https://<host>/realms/MCP/protocol/openid-connect/token", grant_type="client_credentials")

print(token["access_token"])

Include the scope=MCP to ensure custom aud.

✅ 5. Verify Token on Server Side

Use Authlib to validate tokens:

import requests

from authlib.jose import jwt, JsonWebKey, JoseError

jwks = JsonWebKey.import_key_set(requests.get("https://<host>/realms/MCP/protocol/openid-connect/certs").json())

claims_opts = {

  "iss": {"essential": True, "values": ["https://<host>/realms/MCP"]},

  "aud": {"essential": True, "values": ["MCP"]},

  "exp": {"essential": True}

}

def is_valid(token_str):

    try:

        claims = jwt.decode(token_str, jwks, claims_options=claims_opts)

        claims.validate()

        return True

    except JoseError:

        return False

Returns True only if the token is correct, unexpired, properly signed, and intended for your MCP audience.

🛡️ 6. Mitigate Token Leaks

  • Short token lifespan minimizes leak window.

  • Rotate client secret or disable client to prevent future token issuance.

  • Use Not-Before policy (Realm → Tokens) to invalidate all previously issued tokens in emergency.

  • Optionally, enable refresh tokens + revocation for session-aware invalidation.

  • Use token introspection for real-time validation of token status.

🧭 Full Workflow Overview

  1. Create realm MCP

  2. Create confidential client with service-account enabled

  3. Set token expiry at realm/client level

  4. Disable full scope in dedicated client scope

  5. Add Audience mapper to emit aud: MCP

  6. Client fetches token with scope=MCP

  7. Server validates JWT(iss, aud, exp, signature)

  8. Revoke or rotate on compromise + optionally introspect tokens

This setup ensures a clean, secure client‑credentials flow in Keycloak, with tokens that are short-lived and properly scoped for your service.
