This is a real issue I encountered during my linux system upgrade, my ssh connection halt and new connection cannot be rebuild for >5 minutes during yum update.

After system reboot, I got this error message everything I tried "yum update": "problem with installed package kernel-devel-5.4.119-19.0009.54.tl3.x86_64

  - cannot install both kernel-devel-5.4.241-24.0017.14.tl3.x86_64 from BaseOS and kernel-devel-5.4.119-19.0009.54.tl3.x86_64 from @System "

The system reboot breaks the upgrade (sad). if you face the similar issue, follow below thread to resolve.

This issue occurs because there is a conflict between two versions of the kernel-devel package in your system: one installed (5.4.119-19.0009.54) and one you’re trying to install (5.4.241-24.0017.14). The conflict arises when the package manager tries to handle both versions simultaneously.

Steps to Resolve the Issue

1.Identify the Installed kernel-devel Package:

Run the following command to check the currently installed version:

$ rpm -qa | grep kernel-devel

2.Remove the Conflicting kernel-devel Version:

If you’re trying to install the newer version (5.4.241-24.0017.14), remove the older version (5.4.119-19.0009.54) that is causing the conflict:

$ sudo dnf remove kernel-devel-5.4.119-19.0009.54.tl3.x86_64

3.Install the Desired kernel-devel Package:

After removing the conflicting package, install the desired version:

sudo dnf install kernel-devel-5.4.241-24.0017.14.tl3.x86_64

4.Ensure Compatibility with the Current Kernel:

Make sure the kernel-devel version matches the running kernel. To check the current kernel version, run:

$ uname -r

If the kernel-devel version does not match, update your kernel to align with the desired kernel-devel package (can always do this):

sudo dnf update kernel

sudo reboot

In my case, after step 4 without reboot, yum update command has already restore to normal.

5.Clear Metadata Cache (Optional):

If conflicts persist, clear the package manager’s metadata cache to avoid stale information causing the problem:

sudo dnf clean all

6.Reinstall the Correct kernel-devel Version (If Necessary):

If both versions are still causing issues, force the package manager to reinstall the desired version:

sudo dnf reinstall kernel-devel
