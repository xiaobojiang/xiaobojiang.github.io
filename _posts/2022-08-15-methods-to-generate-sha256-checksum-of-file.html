If we download an file or software from the server, we do not want download something that has been tampered by malicious user.

We want to make sure the integrity of the file, SHA256 checksum is wildly applied for this purpose.

We can always find a checksum of the file, at the side of download link.

Figure 1, the SHA256 checksum of RPI download

We can use different commands to help us generate this checksum from the downloaded file, and then compare this with online to verify integrity.

Method 1: Verifying SHA256 checksum with shasum

The shasum command is available on all modern Macs and can be used to check sha256 hash.

Launch Terminal and then use the following command, replacing /path/to/file with the file path as appropriate:

$ shasum -a 256 /path/to/file

For example, to check the sha256 hash of a file called “TopSecret.tgz” in the user Downloads folder, you could use the following:

$ shasum-a 256ubuntu-22.04.1-live-server-amd64.iso

You’ll see something like:

10f19c5b2b8d6db711582e0e27f5116296c34fe4b313ba45f9b201a5007056cb ubuntu-22.04.1-live-server-amd64.iso

Where the string10f19c5b2b8d6db711582e0e27f5116296c34fe4b313ba45f9b201a5007056cb is the sha236 checksum.

Check this with checksum provided by Ubuntu server.

Ubuntu also provided a shasum —check command, in it we can find the checksum string to compare, and it is the same as what we generated.

$ echo "10f19c5b2b8d6db711582e0e27f5116296c34fe4b313ba45f9b201a5007056cb *ubuntu-22.04.1-live-server-amd64.iso" | shasum -a 256 --check

Method 2: Checking sha256 hash with openssl

You can also check and verify sha256 hash by using the openssl command.

From Terminal.app, use the following command:

$ openssl sha256 filename

For example, to verify the sha256 hash of a file named “Data Integrity Matters.pdf” located in the user Documents folder:

opensslsha256ubuntu-22.04.1-live-server-amd64.iso

This will return something like the following:

SHA256(ubuntu-22.04.1-live-server-amd64.iso)= 10f19c5b2b8d6db711582e0e27f5116296c34fe4b313ba45f9b201a5007056cb

With the large string of numbers nd characters being the sha256 hash.

Alright, upper are the two major methods that we can use to generate checksum.
