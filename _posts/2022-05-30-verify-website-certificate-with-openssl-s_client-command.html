It is important to know whether the site is trustable via checking its certificates.

We can view one website’s certificates chain in the browser, eg, I take the screenshot example of bing.com in the Chrome.

Figure 1. certificate chain of bing.com

This gives the information about the chain, who is the signer, and certificate validity.

In the upper picture, Baltimore CyberTrust is the root, the intermediate cert signer is Microsoft, and it signs certificate of bing.com

And the server certificate is valid till September 6, 2022.

There is another way to do it, with openssl s_client command, and it can download the whole server certificate chain.

1, Verify the certificate chain:

$ echo -n | openssl s_client -showcerts -connect <HOST>:<PORT>

echo -n gives a response to the server, so that the connection is released

Try it on bing.com

This gives the same chain as in the Chrome browser.

The depth 2 is the root certificate, depth 1 is the Microsoft intermediate certificate, and depth 0 is the website’s certificate.

And those abbreviations are:

C: country code (IE: Ireland, US: United States)

O: organization

OU: organization unit

CN: common name (for depth 0, it is the website’s full domain name, this helps browser verify that the certificate is assigned to the correct website you are visiting.)

Figure 2. output of openssl s_client command

2, Download the server cert (depth 0) only:

$echo -n | openssl s_client -connect $HOST:$PORTNUMBER -servername $SERVERNAME | openssl x509 > /tmp/$SERVERNAME.cert

That will save the certificate to /tmp/$SERVERNAME.cert.

openssl x509 removes information about the certificate chain and connection details. This is the preferred format to import the certificate into other keystores.

The -servername is used to select the correct certificate when multiple are presented, in the case of SNI.

The x509 at the end will strip out the intermediate certs, you will need to use sed command instead of the x509 at the end.

3, Download the full chain:

$ echo -n | openssl s_client -showcerts -connect <a href="http://home.iotspacex.com:443/">bing.com:443</a> | sed -ne '/-BEGIN CERTIFICATE-/,/-END CERTIFICATE-/p' > ~/Downloads/bing.pem

Here we use -showcerts when we want to download all the certificates in the chain. But if we just want to download the server certificate, there is no need to specify -showcerts.

-showcerts

Displays the server certificate list as sent by the server: it only consists of certificates the server has sent (in the order the server has sent them). It is not a verified chain. 

The openssl or the operating system stored all root certificate locally, so root certificate is not included in the output. 

Now, the certificate of depth 0 and 1 of bing are download into the file.

Note: if the depth is larger than 3, it will recursively download all the chain, except root.

Figure 3. Downloaded example certificate chain of bing.
