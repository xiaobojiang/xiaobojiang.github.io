Handling different Git profiles for various projects doesn’t have to be difficult. With a little setup, you can easily switch between profiles for work and personal projects. Here’s how you can do it usinggitfor work projects andmygitfor personal ones.

Step 1 → Organize Repos by Profile

First, separate the repositories into directories based on their profiles. For example:

•WORK→ for your work-related projects (managed bygit)

•PERSONAL→ for open source or personal projects (managed bymygit)

Step 2 → Set Up Global Git Configuration

Create or edit the global.gitconfigfile in your home directory. This file will tell Git which configuration to use based on the directory path where the repository is located. Add an entry for each profile, as shown below:

[includeIf "gitdir:~/personal/"]
 path= ~/.mygitconfig-personal
[includeIf "gitdir:~/work/"]
 path= ~/.gitconfig-work

This tells Git to use.mygitconfig-personalfor repos in the~/personal/directory and.gitconfig-workfor repos in the~/work/directory.

Step 3 → Create Profile-Specific Configurations

Next, create the individual Git config files for work (~/.gitconfig-work) and personal (~/.mygitconfig-personal) profiles. Customize them with your specific details, like user name and email:

Work Profile (~/.gitconfig-work):

[user]
  name = work_user
  email = work_email@company.com

Personal Profile (~/.mygitconfig-personal):

[user]
 name= personal_user
 email=personal_email@domain.com

Step 4 → Verify the Setup

Now, you have two Git config files (~/.gitconfig-workand~/.mygitconfig-personal). Let’s test that they are applied correctly.

1.For Work:

•Create a new repo in your~/work/directory:

$ cd ~/work
$ mkdir work-test-repo
$ cd work-test-repo
$ git init

•Check the active Git configuration:

$ git config --local --list

You should see your work-specific user details like:

user.name=work_user
user.email=work_email@company.com

2.For Personal:

• Create a new repo in your~/personal/directory:

$ cd ~/personal
$ mkdir personal-test-repo
$ cd personal-test-repo
$ git init

• Check the configuration:

$ git config --local --list

This time, your personal profile will be active:

user.name=personal_user
user.email=personal_email@domain.com

And that’s it! By organizing your repositories and usingincludeIfin your global.gitconfig, Git will automatically use the correct configuration file based on the project directory you’re working in.
