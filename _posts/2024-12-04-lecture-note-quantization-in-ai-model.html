When LLM goes popular, we will hear this word frequently "Quantization", it can help running llama 8B similar models on you laptop, raspberry pi and even on your mobile.

Why Quantization?

  1. Reduced Memory Usage: Lower precision (e.g., int8) requires less memory compared to float32.

• Example: A model with 100 million parameters would require 400 MB in float32 but only 100 MB in int8.

  1. Faster Computation: Operations in lower precision are faster on modern hardware optimized for int8 inference (e.g., NVIDIA Tensor Cores, TPUs).

  2. Energy Efficiency: Lower-precision operations consume less power, which is critical for edge devices.

  3. Minimal Accuracy Drop: With proper techniques, quantized models often retain close to the original accuracy.

What is Quantization?

Quantization is a technique used to reduce the memory footprint and computational requirements of machine learning models by converting their numerical precision from higher precision (e.g., 32-bit floating-point, float32) to lower precision (e.g., 8-bit integers, int8). It is particularly useful during inference, where models are deployed and must run efficiently on resource-constrained devices like smartphones, IoT devices, or even large-scale servers.

scale and zero point are two key parameters that allow the representation of floating-point numbers in lower-precision formats like int8. They ensure that the model can approximate the original floating-point values while working in the integer domain.

Scale (s)

• A floating-point value that defines the step size between quantized integer values.

• It maps the range of integers (e.g., -128 to 127 for signed int8) to the range of floating-point values.

Zero Point (z)

• An integer that aligns the quantized integer zero (q = 0) with the corresponding floating-point zero (x = 0).

• It ensures that zero in the integer domain corresponds to zero in the floating-point domain.

Together, they define the relationship:

x = s * (q - z)

Of source, if we use the scale and zero point to to de-quantization operation, there will be quatization error. We can do calibration after quantization.

Key Components of Quantization

  1. Weights: Convert the model’s learned parameters from float32 to int8.

  2. Activations: Convert input/output values of each layer to int8.

There are few combination of quantization for weights and activation.

one popular is W4INT16, which quantization weight float into 4 bit integer, and activation into 16 bit integer.

Types of Quantization with training.

  1. Post-Training Quantization (PTQ):

• Applied after the model is trained.

• Converts weights and/or activations to lower precision.

• Simple but may lead to a slight loss in accuracy.

• Add calibration step to improve accuracy.

  1. Quantization-Aware Training (QAT):

• Simulates quantization effects during training to help the model adapt.

• More accurate than PTQ but requires retraining.
