I personally like to run jupyter notebook inside a container, it offers flexibility and security isolation to use.

Here’s a ready-to-use podman run command for a persistent Jupyter Notebook setup with sudo access and a podman volumne:

Step 1, create a volume

podman volume create notebookvolume

Step 2, create jupyter notebook container with volume and sudo access

podman run -d \

  --name jupyter \

  -p 10000:8888 \

  -v notebookvolume:/home/jovyan/work:Z \

  -e JUPYTER_TOKEN="change-me" \

  -e GRANT_SUDO=yes \

  -e NB_UID=$(id -u) \

  -e NB_GID=$(id -g) \

  -e TZ=$(cat /etc/timezone 2>/dev/null || echo "UTC") \

  --user root \

  quay.io/jupyter/minimal-notebook:latest

🔑 Key parts explained

  • -v ~/work:/home/jovyan/work:Z Mounts your host’s ~/work into the container, persisting notebooks. :Z makes it SELinux-safe (mandatory on Fedora/RHEL-style distros).

  • -e GRANT_SUDO=yes + --user root Grants the default jovyan user passwordless sudo. The container entrypoint will drop back to jovyan after setup.

  • -e NB_UID=$(id -u) -e NB_GID=$(id -g) Ensures files created in /home/jovyan/work are owned by your host user, not root.

  • -e JUPYTER_TOKEN="change-me" Sets a login token. You can also use JUPYTER_PASSWORD if you prefer a fixed password.

  • -e TZ=... Matches your host’s timezone for correct timestamps in notebooks/logs.

  • --restart-policy=always (Podman equivalent of Docker’s --restart unless-stopped) If you want auto-restart, you can either use this flag or generate a systemd service.

✅ Suggested extras

  • Pin the image version: e.g. quay.io/jupyter/minimal-notebook:lab-4.2.5 instead of :latest.

  • Access logs: podman logs -f jupyter.

Hope you find it useful.
