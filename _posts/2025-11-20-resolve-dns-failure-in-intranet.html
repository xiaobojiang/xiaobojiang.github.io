Recently I found myself cannot visit network in intranet.

The browser complains about DNS issue, IT desk keeps me trying set/unset proxy, with no luck to resolve.

Looking at the error, I thought it should be DNS server issue.

I followed below steps to resolve:

1, find a valid DNS server in intranet. Find your colleague who works well in intranet to get one.

Or, connect to intranet via a working VPN, then use command nslookup <intranet_website_address>

You will get DNS server address in first two lines (one with port 53)


2, add this DNS address into network-DNS server


3, try again into intranet, this issue shall resolve now.
