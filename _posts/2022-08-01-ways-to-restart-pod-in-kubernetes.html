If you are familiar with docker command, docker offers command of restart one container.

However, restart command is not available for Kubernetes.

If you have file change in the pod, or when you upgrade EKS cluster version, and want to restart pod and take effective.

I have two recommended methods for you:

1, kubectl delete pod

Because Kubernetes is a declarative API, the pod API object will contradict the expected one after deleting it, using the command 

$ kubectl delete pod <pod_name> -n <namespace>.

It will automatically recreate the pod to keep it consistent with the expected one.

2, kubectl rollout restart [recommended]

recommended approach is to perform a rolling update of the deployments [1-2] instead of deleting the pods directly as it is more graceful:

$ kubectl rollout restart deployment/<deployment name> -n <namespace>

Replace the placeholder values surrounded by "<>" accordingly. Example for CoreDNS:

$ kubectl rollout restart deployment/coredns -n kube-system

Enjoy.
