In previous post Remote ssh and execute command, I shared how to remote ssh login and execute some command.

Recently, I am trying to prepare some Jenkins pipleline, which will update container image's tag after docker build job is done.

Well, I tried to search a way like k8s to update the container of docker run tag only, but failed to do so.

Finally, I found a way to do this.

First, prepare a script to stop and start container with a specified tag from environment parameter.

Script can be, name is "deploy.sh":

check_docker_pull_errs()
{
  # Function. Parameter 1 is the return code
  # Para. 2 is text to display on failure.
  if [ "${1}" -ne "0" ]; then
    echo "### ERROR ### ${1} : Please make sure image url correct and tag existed on the repo: ${2}"
    # as a bonus, make our script exit with the right error code.
    exit ${1}
  fi
}


docker pull registry.cn-shanghai.aliyuncs.com/XXX/YYY:${APP_TAG}
check_docker_pull_errs $? "registry.cn-shanghai.aliyuncs.com/XXX/YYY:${APP_TAG}"

if [ $( docker ps -a | grep app_deploy_dev | wc -l ) -gt 0 ]; then
  docker stop app_deploy_dev | xargs docker rm ; 
fi 

docker run  -d -m2G --name app_deploy_dev --restart=always -v ~/appdev:/appdev  -p 18080:6000 registry.cn-shanghai.aliyuncs.com/XXX/YYY:${APP_TAG}

Here, suppose we can run this script with proper APP_TAG provisioned, we can make app_deploy_dev container deployed with new tag.

This APP_TAG comes from Jenkins build machine's build running parameter, JENKINS_BUILD_TAG.

Refer to previous script, we can write as:

ssh user@host 'export APP_TAG=${JENKINS_BUILD_TAG};sh ~/.deploy.sh'

This will not work, becuase all command in '' alreadys runs in docker run machine, not in Jenkins server, thus JENKINS_BUILD_TAG is empty and not known on docker run machine.

Let's user xargs

Following is working one:

echo “export APP_TAG=${JENKINS_BUILD_TAG};sh ~/.deploy.sh” | xargs ssh user@host

Use echo command at Jenkins server to output desired shell commands, and pipeline with xargs to run on remote machine via ssh.

An improvement is to avoid host key verification, '-o StrictHostKeyChecking=no'

Final command example is:

echo “export APP_TAG=${JENKINS_BUILD_TAG};sh ~/.deploy.sh” | xargs ssh -o StrictHostKeyChecking=no user@host

That is it. this works perfectly on the machine!
