Introduction

When working with LLMs, free-form text is often good enough. But in production applications, structured data is king.

Imagine building a travel planner: instead of prose like “You should go to Zermatt, Switzerland, for 7 days and try skiing, snowboarding, and après-ski”, you need a validated object like:

{

  "destination": "Zermatt, Switzerland",

  "days": 7,

  "activities": ["skiing", "snowboarding", "après-ski"]

}

This ensures your downstream system — whether it’s a database, API, or UI — gets predictable output.

The challenge? Not all LLMs support structured outputs natively.

  • ✅ OpenAI GPT-4.1 / GPT-4o → support structured outputs with response_format=YourPydanticModel

  • ❌ Qwen3, Llama via Ollama, Mistral → return only raw text, with no schema enforcement

This is where Instructor comes in.

Step 1. The Problem: Qwen3 without Instructor

Here’s what happens if you call Qwen3 directly:

response = client.chat.completions.create(

    model="qwen3-7b-chat",

    messages=[{"role": "user", "content": "Plan a 7-day ski trip in Switzerland."}]

)

print(response.choices[0].message.content)

Output might look like:

"You should go to Zermatt in Switzerland. Spend 7 days skiing, snowboarding, and enjoying Swiss cheese fondue."

Great text, but…

  • No JSON

  • No type safety

  • You’d need brittle regex/parsers to extract fields

Step 2. The OpenAI Way (native structured outputs)

OpenAI models solve this elegantly.

from pydantic import BaseModel

from typing import List

class VacationPlan(BaseModel):

    destination: str

    days: int

    activities: List[str]

response = client.beta.chat.completions.parse(

    model="gpt-4.1",

    messages=[{"role": "user", "content": "Plan a 7-day ski trip in Switzerland."}],

    response_format=VacationPlan,  # native support for schema

)

print(response.choices[0].message.parsed)  # -> VacationPlan(destination='Zermatt', days=7, activities=['skiing', 'snowboarding'])

This works beautifully on GPT-4.1, but not on Qwen3, Llama, or Mistral.

Step 3. Enter Instructor

Instructor is a compatibility layer that makes all LLMs behave like OpenAI in terms of structured output.

Install it:

pip install instructor

Patch your client:

import os

import instructor

from pydantic import BaseModel

from typing import List

#Step 1: Define schema

class VacationPlan(BaseModel):

    destination: str

    days: int

    activities: List[str]

#Step 2: Wrap the client

instructor_client = instructor.from_openai(

    client,

    mode=instructor.Mode.JSON  # enforce JSON mode for models like Qwen3

)

#Step 3: Call the model with schema

def test_qwen3_vacation_plan():

    response = instructor_client.chat.completions.create(

        model=os.getenv("QWEN3_MODEL_NAME"),

        messages=[{

            "role": "user",

            "content": "Plan a 7-day ski trip in Switzerland."

        }],

        response_model=VacationPlan,  # schema enforcement

        extra_body={"chat_template_kwargs": {"enable_thinking": False}},

    )

    print(response)

    print(type(response))  # -> <class 'main.VacationPlan'>

Now, even with Qwen3, you get a validated Python object:

VacationPlan(destination="Zermatt", days=7, activities=["skiing", "snowboarding"])
