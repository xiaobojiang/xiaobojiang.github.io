To set a proxy for container traffic, you can configure the Docker daemon or set environment variables directly on the container. Here’s a short overview of both methods:

  1. Set Proxy for Docker Daemon (Global Setting)

To set a global proxy for all containers, update the Docker daemon’s configuration.

• Step 1: Create or edit a file at /etc/systemd/system/docker.service.d/http-proxy.conf (or /etc/default/docker on older systems):

[Service]
Environment="HTTP_PROXY=http://your-proxy:port"
Environment="HTTPS_PROXY=http://your-proxy:port"
Environment="NO_PROXY=localhost,127.0.0.1,.yourdomain.com"

•Step 2: Restart the Docker service:

sudo systemctl daemon-reload
sudo systemctl restart docker

• Step 3: Restart or recreate containers for the new settings to take effect.

  1. Set Proxy for a Specific Container

You can set proxy environment variables directly when running a container.

• Step 1: Run the container with proxy settings:

docker run -e http_proxy="http://your-proxy:port" -e https_proxy="http://your-proxy:port" -e no_proxy="localhost,127.0.0.1,.yourdomain.com" <image_name>

  1. Set Proxy in Docker Compose

If you’re using Docker Compose, set the environment variables in your docker-compose.yml file.

services:
  my_service:
    image: <image_name>
    environment:
      - http_proxy=http://your-proxy:port
      - https_proxy=http://your-proxy:port
      - no_proxy=localhost,127.0.0.1,.yourdomain.com

•Step 2: Recreate the containers:

docker-compose down
docker-compose up -d

Update the proxy but Container Doesn’t Reflect Daemon Changes Immediately

If you updated the proxy settings for the Docker daemon (e.g., by modifying /etc/systemd/system/docker.service.d/http-proxy.conf), the changes will not take effect for running containers unless you restart Docker and recreate the containers.

• First check general docker daemon information

docker info

• Restart Docker to apply new proxy settings:

sudo systemctl restart docker

• Recreate Containers: If the proxy settings are passed during container creation, you’ll need to recreate the containers to apply the updated proxy settings. This can be done by stopping and removing the container and then starting a new one.

docker stop <container_name>
docker rm <container_name>
docker run -e <other-options> container
